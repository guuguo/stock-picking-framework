## 🔴 公告/新闻搜索（V17.1 新增 — 2026-07-08 案例）

> **来由**: 看到「连续涨停」「异常波动」或「题材异动」时，**必须先查公告/新闻确认原因**，
> 而不是凭「涨了」直接判断。2026-07-07 复盘恒尚节能：6/15 涨停后停牌 15 天，复牌后 4 天 +60%，
> 表面是「妖股」，实际是「**资产重组预案**」。**知道原因后才知该不该碰**。

### 何时触发（强制搜索清单）

```
以下情况出现任一，必须先搜公告/新闻确认原因，再下任何判断：

1. 单日涨幅 ≥ 9.5%（涨停或接近涨停）
2. 连续 ≥ 2 天放量大涨
3. 停牌后复牌
4. 出现「股票交易异常波动公告」
5. 用户/同事提到「妖股」「主力」「抢筹」等情绪词
6. 标的所属行业出现政策/事件突发
```

### 搜索 API 顺序（最快 → 最全）

```
Step 1: 标的存在性确认
  工具: 腾讯 suggest API
  格式: https://suggest3.sinajs.cn/suggest/type=11,12,13,14,15&key={关键词}
  返回: 名称 / 类型 / 代码 / full_code
  用途: 把"国晟科技"等用户口误/笔误纠正为真实标的

Step 2: 公告搜索（结构性事件，优先级最高）
  工具: 东方财富公告 API
  URL:  https://np-anotice-stock.eastmoney.com/api/security/ann
  参数:  stock_list={6位代码}&page_size=10
  返回: 公告标题 + 日期
  重点找: "重大资产重组" / "控制权变更" / "业绩预告" / "风险提示" / "停牌" / "复牌"

  备用: 巨潮公告
  URL:  https://www.cninfo.com.cn/new/hisAnnouncement/query
  注意: 需要 secid 格式（沪市 990002xxxx，深市 xxxxxx），比东财麻烦

Step 3: 新闻搜索（市场情绪 / 行业事件）
  工具: 东财新闻 API
  URL:  https://np-listapi.eastmoney.com/comm/web/getListInfo?code=SH{code}&column=1357
  返回: 媒体 + 标题 + 时间

  备用: 百度搜索 / 同花顺个股页（反爬严，但偶尔能用）

Step 4: 研报（仅用于深度研究，不用于盘中断板判断）
  工具: 东财研报 API
  URL:  https://reportapi.eastmoney.com/report/list?code={code}&pageSize=20
```

## 🔴 锚点≠天花板：仓位决策第一问（V17.1 防呆 — 2026-07-07）

> **来由**: 2026-07-07 四只标的全部误判「已到目标不加」——把首轮锚点当成了仓位天花板。
> 框架原文写的是「强信号可超目标」，但术语「建仓目标」暗示终点，人读到就以为该停。
> 此节为防呆设计：每次判断「能不能加仓」之前，必须先过这道检查。

```
❓ 当前仓位 vs 首轮锚点 vs 天花板(25%)？

首轮锚点: 第一次建仓时的目标区间（中枢上方 10-15%，中枢内部 6-10%，中枢下方 3-5%）
天花板:   单只 ≤25%（硬约束，任何情况不超过）

✅ 仓位 < 锚点下限 → 必须加（连首轮都没建完）
✅ 仓位在锚点区间内 + 新强信号(底背驰/三买) → 可超锚点往 25% 推（重仓层逻辑）
✅ 仓位已超锚点但 <25% + 底背驰/三买再确认 → 可加（趋势确认后二次给信号 = 确定性更高）
⛔ 仓位 ≥25% → 禁止加仓（天花板，无例外）

常见错误: 把「首轮锚点」当成「仓位天花板」→ 仓位到了 10% 就不加了
正确理解: 锚点是起点不是终点，强信号可超锚点但不能超 25%
```

### 实例对照（2026-07-07 复盘）

| 标的 | 仓位 | 锚点范围 | 距天花板 | 错误判断 | 正确判断 |
|------|------|---------|---------|---------|---------|
| 天赐材料 | 3.6% | 6-10% | 21.4% | 「已到目标」❌ | 连锚点下限都没到，必须加 ✅ |
| 恒立液压 | 9.6% | 10-15% | 15.4% | 「已到目标」❌ | 中枢上方+底背驰=重仓层，可追加 ✅ |
| 海光信息 | 6.8% | 6-10% | 18.2% | 「已到目标」❌ | 刚达锚点下限，新信号可往10%+推 ✅ |
| 长电科技 | 15.6% | 10-15% | 9.4% | 「已到目标」❌ | 已超锚点，仍有近10%空间，但需检查盘中顶背驰 ⚠️ |
| 润泽科技 | 15.8% | 6-10% | 9.2% | 「远超锚点不能加」❌ | 中枢内部+底背驰BUY+距天花板9.2%→可加2-3% ✅ |

### 🚨 加仓决策强制检查表（防呆 — 回答「加不加」前必须逐项打勾）

```
每次用户问「XXX能加吗」或你自己判断加/不加，必须按顺序回答：

① 有 BUY / TRIAL 信号吗？
   → 无信号 → ❌ 不加，回答结束。
   → 有信号 → 继续 ↓

② 仓位 ≥25% 天花板了吗？
   → 是 → ❌ 不加（天花板无例外），回答结束。
   → 否 → 继续 ↓

③ 距天花板还有多少空间？（数字，不是感觉）
   → ____% 空间

④ 结论: ✅ 可加，建议加 ____%（不超过天花板空间）。
   
   禁止使用的词汇: 「已到目标」「仓位已满」「锚点上限」「远超锚点」
   正确词汇: 「锚点区间内，距天花板X%，可加」「已超锚点，距天花板X%，信号强可加」
```

## 买入规则

```
触发条件:
  一买: 日线底背驰触发（check_divergence.py 报 🟢 TRIAL）
  二买: 一买后回调不破前低 + 缩量止跌
  三买: 离开中枢后回抽不破中枢上沿 + 缩量止跌
  过滤: 一级主线优先（AI算力/AIDC/半导体），非主线降仓位

分批执行（❌ 不等确认）:
  买点当天: 试错仓，限价单挂在支撑区
    → 不管当天涨幅多少，试错仓都要执行
    → 三买确定性高，试错仓可以比一买大
	  底分型+站上MA5: 加仓到首轮锚点
	    → 这是加仓信号，不是首次买入信号

	仓位三层体系（⚠️ V17 上移：满仓可达 90-100%）:

	  建仓层 — 新信号首次进场:
	    周线中枢上方: 试错 4-5% + 加仓 5-10% = 首轮锚点 10-15%
	    周线中枢内部: 试错 3-4% + 加仓 3-6% = 首轮锚点 6-10%
	    周线中枢下方: 试错 2-3% + 加仓 1-2% = 首轮锚点 3-5%

	  重仓层 — 已有仓位收到新的强信号（底背驰/三买）:
	    中枢上方 + 底背驰再触发: 追加 5-8% → 总仓位可达 15-23%
	    中枢上方 + 三买:            追加 5-7% → 总仓位可达 15-24%
	    逻辑: 趋势确认后二次给信号 = 确定性更高，值得超锚点

	  天花板:
	    单只 ≤25%（硬约束，任何情况不超过）
	    同主线 ≤30%
	    总仓位无硬上限（V17 取消原 60% 隐式上限）

	  满仓场景:
	    全中枢上方三买到位: 6只 × 15% = 90%
	    混合持仓日常:       5-6只 × 各 6-10% = 50-60%

	  ⚠️ 天赐案例（2026-07-01）:
	    → 见上方「🔴 锚点≠天花板」节，此为框架级防呆规则，不再重复

  反弹后追高规则:
    信号当天: 不管涨幅多少，试错仓都要执行（框架原文）
    信号后已反弹 >8%: 不市价追，挂低位限价单在支撑区等回踩
    例: 天赐 49.2→54.5(+10.7%) → 不追，挂 52.00-52.50 等回撤
	  逻辑: 信号当天追是纪律，反弹 10%+ 后追是 FOMO

  口径澄清（语义保持）:
    「限价单挂支撑区」只用于信号当天试错、反弹>8%后等回踩等买入参数。
    一旦已经确认必须成交的加仓/减仓/卖出动作，执行纪律仍是「确认操作 = 市价执行」。
    止损条件单例外，仍按支撑区下方 2% 机械挂单。

	⚠️ 执行纪律：确认就操作，不挂达不到的价（V17 新增）:
	  核心问题:
	    不是差多少的问题——是「确认了要操作，却不去操作」
	    每一笔都挂一个可能达不到的价格 → 反复错过

	  规则（一条就够了）:
	    确认操作 = 立刻市价执行
	    确认买 X 手 → 市价买
	    确认卖 Y 手 → 市价卖
	    不挂达不到的限价单

			  反复踩的坑:
			    黄金 BUY 863 → 挂 850 等 → 没到 → 900 追
			    恒立 BUY 113 → 挂 110 等 → 没到 → 117 追
			    巨人 SELL 顶背驰 → 挂 28.13 等 → 没到 → 27.5 割
			    吉比特 计划 360-363 买 → AI 先说50股(不可买)且确认太慢 → 365.65 买100股 → 多付约 265 元

			  ⚠️ 最小交易单位 + 一手金额 + 价格容忍带（V17.2 新增 — 2026-07-08 吉比特案例）:
			    这是回答「能不能买/加」前的 P0 检查，必须先做，不能先给不可执行的 50 股建议。
			
			    A股买入执行单位:
			      买入默认必须 100 股整数倍（科创/北交特殊规则另查，但默认先按 100 股）
			      卖出可处理零股；买入不要建议 50 股/30 股这类不可执行单位
			
			    回答买入问题的强制顺序:
			      1. 先算一手金额 = 现价 × 100
			      2. 算一手金额 / 总资产，占比多少
			      3. 判断这个最小仓位是否符合当前买点级别
			      4. 再给「买 100 股 / 不买 / 等」的结论
			
			    一手金额风控表:
			      ≤3% 总资产: 可作为轻仓试错
			      3%-6%:      可作为正常试错/补仓（如吉比特 365×100≈3.6万≈3.5%）
			      6%-10%:     需要强信号 + 高评分 + 领先指标支持
			      >10%:       不适合试错（如中际旭创 1155×100≈11.5万≈11%）
			
			  ⚠️ 价格容忍带（防止“差几块不买，追高多亏”）:
			    一旦框架结论是「买最小单位」且现价距离计划买点 ≤1.5%，必须立刻给执行结论。
			    不允许因为想多等 0.5%-1% 的便宜价而放弃动作，也不允许回答太慢导致用户追高。
			
			    规则:
			      计划买点价 P，现价 C
			      若 C <= P * 1.015 且买入理由未变 → ✅ 立即买最小单位（通常100股）
			      若 C >  P * 1.015 → 🟡 挂 P 附近，不追
			      若买入理由变坏（业绩/公告/背驰失效）→ ❌ 不买
			
			    核心: 差 1% 不是风险，仓位过大才是风险；买点成立时，执行速度比省小钱重要。

			⚠️ 试错仓必须追到首轮锚点（V17 新增）:
		  反复踩的坑:
		    浪潮: 试错 2.7% → 确认了 → 没加 → 涨了和没涨一样
		    黄金: 试错 1.4% → 确认了 → 讨论三天 → 仓位太轻
		    吉比特: 试错 3.2% → 没确认 → 不该加 → 加了

		  规则:
		    每次盯盘检查: 持仓股是否已确认 + 仓位 < 首轮锚点下限
		    → 如果有，直接加到锚点区间中位
		    → 不等新信号，确认了就是信号

		  检查表（每次盯盘过一遍）:
		    润泽 19.5% vs 锚点 10-15% → 已超锚点，距天花板(25%)还有 5.5%，有新强信号可加
		    浪潮 2.7%  vs 锚点 6-10% → 🚨 确认+仓位不足 → 加！
		    黄金 1.4%  vs 锚点 5%    → 🚨 确认+仓位不足 → 等黄金二买

	⚠️ 卖出纪律：不等反弹（V17 新增）:
	  规则:
	    确认要卖 → 市价卖
	    不挂达不到的限价单
	    不分「新鲜信号等反弹」「陈旧信号现价卖」——确认了就执行
	```

## 卖出规则

```
减仓（顶背驰触发）:
  顶背驰当天 → 挂回落卖出（监控当天高点，回落 3%）
  当天没触发   → 次日开盘市价卖 1/3 或一半
  不等反弹，不追杀，不犹豫

  周线中枢上方 → 只减 1/3，不动主仓
  周线中枢内部 → 减 1/2
  周线中枢下方 + 顶背驰 → 全清

止损（机械执行）:
  两档止损，不一刀切:
    减压位: 跌破短期支撑 → 减 1/2，剩下一半等反弹不补仓
    止损位: 跌破结构支撑 → 市价全出，不等不看不扛
  止损出局后 T+14 不追回

⚠️ V17 修正: WAIT 不追杀 → 次日市价卖出
  顶背驰当天未卖且已跌 >5% → 次日开盘市价卖一半
  不再等反弹。差几百块不重要，卖掉比卖高价重要。
  案例: 巨人 29→27.5 等了三天越等越低，工业富联 79→66 同理

止损位设定规则:
  一买持仓: 前低下方 2%
  二买持仓: 一买低点下方 2%
  三买持仓: 中枢上沿下方 2%
  没有买点类型的持仓（左侧潜伏）: 前低下方 2%

减压位设定规则:
  跌破 MA20 或成本线附近 → 减 1/2
  比止损位提前 3-5% 预警

🔴 DANGER 信号处理（⚠️ V17 新增自动解除）:

  DANGER = 下跌放量确认（价跌量增，下跌动力未耗尽）

  对观察池: 远离，不接刀
  对持仓:   全清，不等破止损

  自动解除（满足任一即可）:
    条件1: 自 DANGER 触发以来，价格反弹超过触发低点的 5%
    条件2: DANGER 触发超过 7 天，且期间未创新低

  解除后: 底信号恢复正常的 BUY/TRIAL detection
  代码: chanlun_signals.py → resolve_stale_danger()
```

## 盘中节奏

```

日线买点出现:
  盘中探底回升/缩量止跌 → 直接挂限价单，不等尾盘

日线卖点出现:

  盘中稳步上涨 → 等尾盘，收盘价确认后再挂

日线无信号:
  不动。不靠盘中走势开新仓，不靠盘中走势清仓。
```

### 矩阵读取规则

- **盘中决策优先看「今日走势」列**（日线是历史，今日走势是当下）
- **现价/今日涨跌用实时价算**（日K未收盘数据不准）
- **信号 ≤5 天有效期**，买卖冲突取最新
- **今日涨跌 ≠ vsMA5**，盯盘汇报必须区分
- **换仓/新买必须先跑矩阵**，输出周线×日线对比结论

### 信号冲突解决（⚠️ V16.1 新增，已写入 check_divergence.py）

```
当同一标的同时存在买卖信号时，按以下优先级解决：

① 漏卖规则（代码层 apply_missed_top_rule）:
   判断标准: 信号以来的最低价（非当前收盘价）是否跌离高点 >5%
   触发后: SELL → 🟡 WAIT（状态降级为提醒；执行口径以 V17「次日市价卖出」为准）
   关键: 用最低价而非收盘价，防止反弹后误判信号仍有效

② 买卖冲突 → 取最新日期:
   同一标的既有 SELL 又有 BUY → 比信号日期
   底背驰日期 ≥ 顶背驰日期 → 抑制 SELL，保留 BUY
   顶背驰日期 > 底背驰日期 → 抑制 BUY，保留 SELL

③ 信号过期（>5天自动失效）:
   信号日期距今 >5 天 → 自动过期，不输出操作建议
   WARN / WAIT / WATCH 级别不过期（仅提醒，不触发操作）

④ 透明化输出:
   被抑制的信号在 🔇 自动过滤段落展示，含抑制原因
   避免静默过滤导致用户不知道发生了什么
```

**天赐材料案例（2026-07-01）：**
```
顶背驰 SELL (6/24, 56.9) + 底背驰 BUY (6/29, 49.2)
  → ① 漏卖规则: 最低价 49.2 vs 高点 56.9 = -13.5% > 5% → SELL→🟡WAIT
  → ② 冲突解决: BUY(6/29) > SELL(6/24) → 抑制 WAIT
  → 最终: 只输出 🟢 BUY ✅
```

## 与 chanlun-analysis 的分工

| 维度 | chanlun-analysis | stock-picking-framework |
|------|-----------------|------------------------|
| 定位 | 结构分析（笔、中枢、背驰判断） | 决策执行（仓位、条件单、盯盘） |
| 买点识别 | 教你怎么找一买/二买/三买 | 找到买点后怎么分批买 |
| 仓位 | 决策矩阵（+4-6%等） | 具体首轮锚点（6-10%等）+ 天花板25% |
| 条件单 | 原则（背驰当天行动） | 具体挂单参数 |

**冲突时 SPF 优先。chanlun 是分析方法，SPF 是执行纪律。**
