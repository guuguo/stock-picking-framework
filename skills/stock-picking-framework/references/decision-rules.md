# Decision Rules 详解 (Layer 2: R0-R23 + 执行纪律)

> 何时加载: 真正要做研究 / 加减仓 / 挂单 / 复盘, 需要某条 R 规则的**完整执行细节**时。
> SKILL.md 只保留 5 原则 + R 规则一句话索引; 本文件是可按需展开的操作手册。
> 最高优先级: 任何研究 / 加仓建议必须先走这里的 R0-R22。

## Layer 0: 公理 (Axioms)

```
A1 [目的] 炒股的目的是长期最大化期望收益,
          不是"挂到股票" / "踏中风口" / "证明判断对"

A2 [决策] 决策依据 = 期望年化收益 × 触发概率 (期望贡献),
          不是单一价格点位或单一估值

A3 [约束] 行动幅度有硬上限, 单次决策不致命, 累积纠错可持续
```

## P1 证据 (Evidence)

### R1 [估值] FCEM 四源交叉验证, 不盲信券商
- 任何 Y+2 净利估算 → 量价利 + TAM × 市占率 + 历史回归 + 同行映射 4 方法
- 散度 ≤ 30% 才高置信 (详见 `forecasting-model.md`)
- 券商一致预期仅作 sanity check, 不作 anchor (尤其季报后 > 3 月未更新)

### R1b [自主推演 vs 一致预期] 历史用一手, 未来自己推

> 案例驱动: V3 用 simplywall 一致预期 +21.5% 当 Base 锚, 违反"自主推演"原则。

```
数据来源分层 (严格遵守):

Layer A: 历史数据 (一手, 不靠机构二手)
  ✅ 必查源: a-stock-data 直连源 (巨潮公告/新浪财报/东财/腾讯) / 公司年报 PDF; akshare 仅作 fallback
  ❌ 禁用: 二手新闻 (moomoo / quartr / simplywall)
  执行: R0 跑 data_fetcher.py 拉真实季度数据

Layer B: 未来数据 (自主推演)
  ✅ 必做: R2 7 步可持续性分析推演 Bear/Base/Bull
  ✅ 量价拆解, 不能 Q1 × 4 粗暴
  ✅ 基数效应推演 (Q1/Q2/Q3/Q4 + 下一年)

Layer C: 一致预期 (仅 sanity check)
  ✅ 用途: 自主推演 vs 券商中位数 偏离检查
  ❌ 禁用: 直接当 Base 锚

偏离规则:
  偏离 < 30%: 通过, 维持自主推演
  偏离 30-50%: 标 R5 散度高, 加大 Bear 概率, 重新检查方法
  偏离 > 50%: 必须重检 (要么自己错, 要么市场错)

实证: 乐鑫 V3 (5/26) 错用 simplywall 当锚 → 价值锚 ¥146 偏低
     正确: R0 拉一手数据 + R2 自主推演 → 价值锚自主估算
```

### R2 [可持续性] 任一指标 yoy ≥ +30% 必做 7 步可持续性分析
- 基数推演 / 量价拆解 / 订单可见度 / Capex 节奏 / 竞争格局 / 三档情景 / 证伪信号
- 详见 `deep-research.md` 维度 6
- 违反 → 评分至少高估 1-2 分, 必须重做

### R3 [财报查证] 最新季报必查 + 双校验 + 评分对冲
- "利空出尽 vs 业绩证伪": 营收 ≥ +10% + 扣非负 = 投入期; 营收 < +5% + 扣非负 = 失速期
- 三层利润必列最新季 + 同比 + 环比, 警惕反转 ≥ 20pp
- 同行业横向校验, 防止把行业 β 当公司 α

### R3.1 [三层利润 → 评分定档] 基本面底座 (维度 2) 必须用最深一层定档

> 案例驱动: 易点 V1 三层利润写了"深层 +12.45% 主营稳健 ✅", 但维度 2 评分仍打 5.0 → 评分体系与三层利润洞察"断头桥"。
> ⚠️ 完整的"会计幻觉过滤器" 5 步流程见 `deep-research.md` 维度 2.5, 不在此重复。

```
评分前必须先按"三层利润最深层"定基本面底座 (维度 2) 上下限:

  深层利润 yoy (剔除汇兑 / 股份支付 / 公允价值变动 / 一次性投资收益):
    > +20%:  维度 2 ≥ 7.5
    +10~+20%: 维度 2 ≥ 7.0
    +0~+10%:  维度 2 ≥ 6.0
    -10~0%:   维度 2 ≥ 5.0
    < -10%:   维度 2 < 5.0 (允许低分)

  且必须同时通过:
    □ 经营现金流 yoy 不能严重恶化 (≥ -50%)
    □ 应收/存货周转不能急剧恶化 (≥ -30%)
    □ 毛利率同比 不能塌方 (≤ -3pp)
    任一失守 → 维度 2 上限 -1.5 分

特殊项识别 → 评分对冲 (核心规则):
  若 表层归母 vs 中层扣非 vs 深层 利润 差距 > 50%:
    → 必走 R3 5 个为什么追到根因
    → 若 根因 = 一次性 + 已识别 + Q+1 大概率反转:
      ✅ 维度 2 (基本面) 用 深层利润 评分, 不用归母
      ✅ 维度 3 (预期差) 加分 (+0.5~1): 一次性反转就是预期差
      ✅ 维度 5 (催化集群) 加分 (+0.5~1): Q+1 冲回是催化
      ✅ 维度 7 (持续性) 看主营趋势, 不被一次性影响
      ❌ 不允许同一一次性事件被维度 2/3/6/7 重复扣分

实证 (易点 V1): 评分 6.35 (按表层重复扣分) → 7.40 (按深层 + 对冲), 与 V9 7.6 连续
```

### R13 [Forward PE 三年视角] 不单看 PE(TTM), 必出 Y+1/Y+2/Y+3 fwd PE 表
- TTM PE 高 ≠ 高估, 高成长股 PE(TTM) 永远看似高 (兆易 75x / 润泽 68.7x)
- 主决策看 Y+1 fwd PE 与价值锚同源 (券商一致预期 × PE)
- 评级按行业类型分 4 档 (A 周期 / B 稳定 / C AI 高成长 / D ODM), 详 `forecasting-model.md` 七
- 实证: 润泽 V1 用 TTM PE 68.7x 误判 "全档期望负" → V1.2 发现 Y+2 fwd PE 33.7x 是定价 Base

### R14 [bottom-up 实证] 任何 Y+2 估算前必拉 ≥ 3 份券商研报 PDF + 提取业务驱动数据
- 拒绝 "估" 字开头的量价利推演, 必须是 "实证出处 + 公告页码"
- 4 个核心问题答不出 → 不能出 Bull 估值:
  1. 当前装机/产能, 在建, 何时投产?
  2. 锁单率 / 6月12月可见度?
  3. Top 3 客户占比 / Capex 指引?
  4. 单 MW / 单订单实际收入 (财报倒推)?
- 行业 PE 中枢分级表 (R14.1, 7 类) 见 `forecasting-model.md` 二
- 实证: 润泽 V1 漏掉 240/420 MW 锁单 + REIT 上市 + 40kW 液冷 → V1.2 补全后价值锚 1.10→1.05
- 脚本: 通用抓取 `scripts/core/fetch_stock_data.py <ticker>` (同行配置 `scripts/core/stocks_config.json`); 研报 PDF 下载/解析见 `references/deep-research.md` 维度 0

## P2 概率 (Probability)

### R4 [情景] 任何预测必须 Bear/Base/Bull 三档 + 显式概率
- 默认 25/50/25
- 调整: Bear 证伪 ≥ 2 个 → Bear +10%; Bull 超预期 ≥ 2 个 → Bull +10%; L1 黑天鹅 → Bear +20%
- 任何研究输出必含三档表

### R5 [散度] FCEM 四方法散度 > 30% → 标记不确定
- ≤ 20%: 高置信, 用中位数
- 20-30%: 中等, 加大 Bear 概率到 30%
- 30-50%: 战略档不挂 (仅战术档)
- > 50%: 检查并剔除最不适用的方法, 重新验证

### R15 [同行 Q1 alpha 判定] Bull/Bear 概率调整的硬证据
- 必出同行 3-5 家最新季营收同比对比表 (业务相似度 > 60% 才可比)
- 标的同比 − 行业均值 > +20pp → 强 alpha → Bull 概率 +10-15pp
- 标的同比 − 行业均值 < -10pp → alpha 损失 → Bear 概率 +10pp
- 实证: 润泽 Q1 +53.55% vs 光环 -10.86% / 数据港 -3.80% = +60pp → Bull 30% → 45%
- 详见 `deep-research.md` 维度 5

### R16 [账户级浮亏护栏] 单只仓位上限随账户健康度自动收缩

> 设计动机: 25% 单只破格上限是"账户健康"前提下的设计。一旦账户已亏, "理性时刻填表能扛"会被"恐慌时刻真实情绪"击穿 (hot-cold empathy gap)。此护栏与单只仓位上限独立, 是 fail-safe。

| 账户整体浮亏 | 自动动作 | 单只破格档上限 |
|---|---|---:|
| 0 - 10% | 正常 | 25% (核心档) |
| 10 - 15% | 暂停所有新建仓 24h | 22% |
| 15 - 20% | 触发减仓: 破格档 → 普通档 | 18% |
| 20 - 25% | 试错档全部清空, 普通档减半 | 12% |
| ≥ 25% | 入"冷静期" 1 周不交易 | 全员降级 |

触发执行原则:
- 账户级浮亏看**周末收盘**, 不看盘中波动
- 触发后 T+1 开盘自动执行降级 (不犹豫不查证)
- 恢复需账户回升至上一档 + 持续 2 周

实证锚点: 2026-04 大盘事件, 账户实际浮亏 ¥220k (~20%) 时真实反应 = 失眠 / 想清仓。此为锚点, 不可假装"那次能扛"。

## P3 对称 (Symmetry)

### R6 [买卖对称 + 让赢家跑] 买入门槛 ≤ 卖出门槛; 赢家默认持有, 卖出要更高门槛
- "4 项准入"仅用于建仓, 不用于已持仓加码
- 已持仓加码只看触发位 + RSI 不在 A 区
- 实证 (2026/5): 润泽 ¥95 / 兆易 ¥330 因买严错过 → 此后修正

> 让赢家跑: 框架最贵的系统性错误是"砍掉赢家"。中际 ¥550→¥1037 (漏 ¥97k) / 兆易 ¥330→¥492 (漏 ¥32k) 都是涨幅最大的票被规则赶走。A 股动量牛市里, 少数大赢家贡献组合大部分收益。"涨了就想兑现"是处置效应, 是亏钱根源。

```
赢家处置规则 (持仓已浮盈的标的):

1. 默认动作 = 持有, 不是兑现。卖出赢家的门槛要 > 买入门槛。
   只有以下之一成立才减赢家:
     ✗ 基本面证伪 (业绩反转 / 主线被推翻, 走 R3)
     ✗ 估值进入 > 2.0 严重高估区 (R10)
     ✗ 技术面双极端超买 (RSI6>80 + RSI14>70, R12 A 区) 且需要现金
     ✗ 单只 > 仓位上限需再平衡 (R11)
   "涨了 X%, 落袋为安" / "怕回吐" 不是减仓理由 (R20 反面)。

2. 移动止盈代替固定止盈点:
   - 不再预设"¥X 卖一批"的固定档 (那等于给赢家设天花板)
   - 改用回撤触发: 从持仓期最高点回撤 N% 才动
       核心/普通档: 回撤 18-20% 减半
       试错档:      回撤 15% 减半
   - 最高点上移 → 止盈位跟随上移

3. 分批兑现只在"极端超买 + 估值严重透支"双杀时启动, 每批 ≤ 1/3 (R7)。

4. 加仓赢家 (金字塔正向) 允许, 但只在回调到触发位 + RSI 非 A 区 + 不破集中度上限时。

看板脚本: python3 scripts/core/winners_guard.py (移动止盈 + 集中度提示)
```

### R6b [止损对称 — 赢家也要保护利润]
- 让赢家跑 ≠ 放任回吐到亏损。移动止盈位一旦上移到成本之上, 就是"保本/保利"硬线
- 与 R12 双触发止损联动: 赢家挂"移动止盈"(跟随最高点), 不挂"固定止盈"

### R6c [边界澄清] "让盈利奔跑" vs "提现金减仓" 是两个独立决策

> 动机: "不是说让盈利奔跑吗, 怎么又说赚钱的先卖?" 本条划清边界。

```
问题 A "要不要卖这只赢家?"  → 默认: 不卖 (R6)。只有 R6 那 4 个条件之一成立才卖。
问题 B "我已决定要腾资金, 该从谁手里拿?"  → 见下方优先级

腾资金的标的选择优先级 (从高到低):

第 1 顺位: 退最弱的整只 (同时服务"减到 6 只" + "补现金")
   - 评分 × 期望贡献 末位的弱票, 整只清掉, 完全不碰赢家。首选。

第 2 顺位: 削最热的票一部分 (仅当不愿动弱票, 且确实要防崩盘补现金)
   - 必须同时满足赢家处置例外: RSI 双极端 (R12 A 区) + 需要现金
   - 量级 ≤ 持仓 1/3 (R7), 只削不清。属下策。

第 3 顺位: 绝不动 — 没涨的低位票 / 健康奔跑中的赢家。

关键认知: "提现金"是可选动作 (为防崩盘 / 修复护栏), 不是默认动作。
  不缺现金、护栏没破 → 一只都不卖。缺现金 → 优先退弱票, 而不是削赢家。
```

### R6d [评分减仓联动] 评分持续下调 → 强制减仓 (V14.4 新增)

> **动机**: 三七评分从 8.05 降至 7.3 再降至 7.0, 全程没有一次主动减仓, 最终被止损暴力清仓 (9700股一次性出清, 亏¥34k)。
> 评分下调 = 基本面恶化信号, 但框架此前没有"评分→仓位"的联动机制。

```
评分减仓触发 (绝对阈值, 不看相对排名):

  单版本下调 ≥ 0.5 → 仓位上限降一档 (核心→普通→试错)
  连续 2 版本下调  → 减至普通档上限 (8%), 不论相对排名
  评分跌破 7.0     → 自动进入退出候选, 下个调仓日执行
  评分跌破 6.5     → T+3 交易日内必须减仓至 ≤ 5% 或清仓

执行窗口: 不要求立刻执行, 给 3 个交易日找合适价位。但窗口期内禁止加仓。

降档对账 (强制, 防"改了上游忘下游"):
  降档 = 上游决策 (仓位目标↓), 但止损价/止损量是下游参数, 必须连带复核:
    □ 止损"量"是否仍覆盖降档后的剩余仓? (不能留旧止损量, 否则部分仓裸奔)
    □ 止损"价"是否该随评分收紧? (旧止损常是"看好留底仓"时代设定, 降档后该上移)
    □ "留底仓看下一季"逻辑是否还成立? 若降档理由本身=下一季证伪风险 → 该逻辑作废, 不留裸仓
  实证 (巨人 2026-06-24): 评分8.0→7.0降档12%→8%(减1700→剩3300), 但旧止损22.80/1600股
    是评分8.0"留底仓看Q2"时代设定 → 剩3300只保护1600=1700裸奔。
    且降档理由正是"Q2流水峰值证伪" → "看Q2"逻辑作废。
    修正: 止损上移23.40(一买失效), 3300全保护, 不留裸仓。

与 R6c 的关系:
  R6c 解决 "该退谁" (相对排序, 退最弱的)
  R6d 解决 "这只该减多少" (绝对阈值, 不看别人)
  两者独立, 可能同时触发: R6c 退出最弱票 + R6d 减恶化票

实证对照:
  三七 8.05→7.3(-0.75): 触发"仓位上限降一档", 17.4%应降至≤12%
  三七 7.3→7.0(连续2版降): 触发"减至8%", 应减至780股
  如果 R6d 存在, 三七不会扛到9700股止损
```

### R7 [量级对称] 减仓量级 ≤ 加仓量级 × 1.5
- 单笔金额: 减仓 ≤ 同一周期内最大单笔加仓金额 × 1.5
- 实证 (2026/5/11): 平高减 3000 股 ¥69k 违反 → 改 1500 股分两批

### R8 [配对] 减仓必同步配套加仓预案 ("释放-部署")
- 减仓 X 元 → 同时给 ≥ 1 个核心标的接盘点位 + 量级
- 释放资金 ≤ 7 天内有明确部署目标
- 例外: 主线全 A 区双极端时, 不强行减非主线

## P4 期望收益 (Expected Value) ⭐ 核心

### R9 [仓位分配] 仓位按"期望年化 × 触发概率"排序

```
每个候选挂单价 → 计算:
  期望年化收益 = Σ (情景概率 × 价位回报)
  触发概率 = 1 年内被触发的可能性
  期望贡献 = 期望年化 × 触发概率

仓位规则:
  期望贡献 > 6%: 主力仓 (5+ 手)
  期望贡献 > 3%: 战术仓 (2-3 手)
  期望贡献 > 1%: 心理仓 (1 手, 维持选择权)
  期望贡献 < 1%: 不挂

不为"全档位覆盖"稀释弹药 / 不为"挂到"买高估值 / 不为"价值底"放弃主线敞口
```

### R10 [双锚定] 战略档 = max(价值锚 × 0.85, 现价 × 0.80)

- 价值锚 = Y+2 期望合理价 (FCEM 加权), 仅作认知工具, 不挂单
- 战略档实际挂单, 兼顾"价值底线"与"现实触发概率"
- 主线溢价上限 = 价值锚 × 1.5, 现价超此线 → 战术档减半

估值区间分级仓位 (与 R17 联动):

| 现价/价值锚 | 区间 | 仓位策略 |
|---|---|---|
| < 1.0 | 低估 | 加仓优先级最高, R11 允许打满档位 |
| 1.0-1.2 | 合理 | 持有, 小幅加仓 |
| 1.2-1.5 | 主线溢价 | 战术敞口 (普通档), 不进核心 |
| 1.5-2.0 | 高估但仍可博弈 | 强主线 + 业绩硬时, 仍允许试错档 4% 占坑 (中际 ¥550 教训) |
| > 2.0 | 严重高估 | 禁入 (兆易 1.99 临界已禁) |

关键认知: 1.5-2.0 区间不是"完全空仓"的理由。拒绝该区间强主线标的 = 错过全部行情 (中际 ¥550 → ¥1,037 漏 ¥97k)。

挂单档位表:

| 档位 | 锚定 | 距现价典型 | 量级 | 角色 |
|------|------|----------:|------|------|
| 起仓 | 现价 × 0.97 | -3% | 1 手 (100 股) | 心理仓 (避免 FOMO) |
| 主仓 | 现价 × 0.92 | -8% | 2-3 手 (200-300) | 战术加码 |
| 战略 | max(价值锚×0.85, 现价×0.80) | -15~-25% | 主力 3-5 手 (300-500) | 期望贡献最高 |
| 极端 (可选) | 价值锚 × 0.8 | -35~-50% | 1-2 手 | 黑天鹅备份 |

核心认知: 价值锚 ≠ 战略档。价值锚是"认知温度计", 战略档是"挂单价"(兼顾触发率)。详细战略档计算见 `forecasting-model.md` 七.5。

## P5 约束 (Constraints)

### R11 [仓位上限]
- 同一主线 ≤ 30% 总仓
- 单只标的: 核心 25% / 普通 12% / 试错 4%
- 持仓数量 ≤ 6 只 (理想 4-5); 满 6 只时"新进必先退", 退评分×期望贡献末位
- 前 3 大持仓 ≥ 60% 总仓 (资金向高信念集中, 治"伪集中")
- 核心档准入: V14 6 选 5 (见 SKILL.md Position Sizing, 已删除"持有 3 月" proxy 约束)
- 低估区自动打满档位: 现价/价值锚 ≤ 1.0 时, 不留"档位上限缓冲", 直接到达上限
- 单次买卖 ≤ 持仓 1/3 (强制分批)
- 试错档总和 ≤ 10%, 现金 ≥ 5%
- 评分 < 7 不能进主仓位档 (仅试错 4%)
- 自检: 现金 > 25% 持续 1 周 → 检查买入纪律是否过严
- 6 个月内不准修改仓位上限数字本身

### R12 [执行约束]
- A 股: 100 股整数倍, 卖出不足 100 股一次清掉
- A 区双极端 (RSI6 > 80 + RSI14 > 70): 禁止买入
- 双触发止损: 上破止盈位 + 下破止损位 同时挂 (避免阴跌折磨)
- 到点必买: 预设观察价触发, 至少 1 手起仓 (避免 FOMO 后悔)
- 决策成熟即执行: 节省金额 < 错过概率 × 错过损失 → 直接市价

### R17 [起仓档双过滤] 基本面 + 技术面 双层判断, 不允许"等大回调"思维

> 案例驱动: 中际旭创 ¥550 错过 ¥97k。错误模式: 仅凭"PE 偏高 / 价值锚 > 1"就建议"等 -10% 回调"。

```
建仓位决策 = Layer 1 基本面过滤 × Layer 2 技术面过滤 × Layer 3 执行

Layer 1: 基本面是否过关? (主线 + 业绩 + PE)
  - 主线强 (新质生产力前 3 大) + 业绩硬 (Q1 yoy ≥ +30% 或 周期反转龙头先行) + PE 合理 (Y+1 fwd PE < 50x 或 PEG < 1)
  → 不过关: PASS, 不进观察池;  过关: 进 Layer 2

Layer 2: 当前技术面是否过热?
  - RSI(6) > 80 + RSI(14) > 70: 双极端超买, 禁买 (R12)
  - 距 60 日均线 > +20% / 近 7 日累涨 > 15% / 近 6 周累涨 > 60% / 当日涨幅 > +5% 或涨停
  → 任一触发: 等回调到温和区 (RSI 60-75) 起仓;  都没触发: 进 Layer 3

Layer 3: 起仓档与执行 (沿用 R10 + R12)
  - 起仓档: 现价 -0.5% ~ -1% (保证成交), 不允许 -3% 以下
  - 量级: 100 股小试占坑
  - 决策成熟即执行: 节省 < 错过概率 × 错过损失 → 直接市价

禁止行为: 仅凭"PE 高"等 -10% 回调 / 仅凭"价值锚 > 1"放弃强主线 / 起仓档限价 ≤ -2%

案例对比:
  中际 4月 ¥550: L1 ✅ + L2 ✅ (RSI 中性) → 应直接 ¥545 起仓 100 股
  中际 5/22 ¥1,037: L1 ✅ + L2 ❌ (6 周 +116%, RSI 85+) → 等回调 ¥850-900
  兆易 5/22 ¥468: L1 ❌ (PE 150x + Q1 -12% 无业绩) → 永不入场
```

### R18 [挂单类型选择] 按触发概率分配挂单形式

> A 股实际机制: 没有 GTC, 只有"当日限价单 + 条件单 + App 提醒"。误用条件单做今日大概率触发的单, 1-3 秒报价延迟错过尖顶 (乐鑫 ¥194.5 案例)。

```
| 触发概率 | 推荐类型 | 优劣 |
|---|---|---|
| 今日 > 30% | 当日限价单 | 提前进交易所排队, 时间优先, 收盘失效需重挂 |
| 今日 < 30%, 1-4 周内 | 条件单 (本地) | 多日有效, 但触发后报入有 1-3 秒延迟 |
| > 1 月不会到 | App 价格提醒 | 永久, 触发后人工决策 |

条件单设置原则:
  - 触发价 ≠ 委托价, 错开 ¥0.5-1
  - 卖出: 触发价 < 委托价 (例: 触发 ¥194 委托 ¥194.5)
  - 买入: 触发价 > 委托价 (例: 触发 ¥66 委托 ¥65.5)
  - 目的: 提前报入限价单, 避免尖顶瞬间触发但来不及成交
```

### R19 [决策频率约束] 盯盘可以, 决策慎重

> 案例驱动: 5/21 上午 9 次刷盘 + 1 次手动加仓后 R11 进一步恶化。核心: 不约束信息收集 (盯盘), 只约束行动 (决策)。

```
盯盘: 不限次数 (只要不操作, 无副作用)

决策约束 (任一买入/卖出/撤改单前必查 5 项):
  ✅ 现金充足 (R11: ≥ 5%)
  ✅ 单日浮盈/亏 < ¥16k (R7 警戒线)
  ✅ 该标的有研究依据 (V14 评分 + FCEM 价值锚)
  ✅ 触发档位明确 (不是临时拍脑袋)
  ✅ 量级合规 (单笔 ≤ 持仓 1/3, 加仓 ≤ 加仓量级 × 1.5)

任一不满足 → 不操作, 但盯盘可继续;  全部满足 → 执行

AI 助手协议:
  "盯一下" → 给数据, 不主动催操作
  "我想加仓" → 用 5 项过滤
  用户已决定操作 → 执行 + 记录, 不阻拦
```

### R20 [沉没成本绝缘] 任何决策不引用过去

> 案例驱动: 兆易 ¥330 + 中际 ¥550 错过反复出现"修复"冲动。沉没成本 (亏损方向) + 错过收益 (利润方向) 是同一谬误的两面。

```
禁止以以下理由做决策:
  ❌ "我之前 ¥X 就该买的, 现在追"        (错过修复)
  ❌ "我已经亏 ¥X 了, 不能割"            (沉没成本)
  ❌ "已经研究 X 小时了, 不能不买"        (时间投入)
  ❌ "建仓成本 ¥X, 跌到 ¥X 之下才止损"   (锚定成本)

每次决策只看当前: 基本面 / 技术面 (R17 L2) / 现金仓位 (R11) / 期望值 (R10) / 止损位 (Bear 价 × 1.15, 与成本无关)

判断"现在该不该买/卖"的最有效方法:
  假设你从来没研究过这只股票, 不知道它涨/跌了多少, 不知道你赚/亏多少
  只看现在的基本面 + 技术面 + 估值 + 仓位 + 现金 → 你会买/卖吗?
  会 → 执行; 不会 → 不执行。过去的错过 / 亏损 / 投入 一律忽略。
```

## 执行纪律细则 (R12 展开: 决策成熟 → 执行果断)

### 核心原则: 值得买就直接买, 不为蝇头小利冒踏空风险

| 阶段 | 状态 | 原则 |
|------|------|------|
| 决策阶段 | "要不要买" | 慢, 反复验证, 纪律对抗 FOMO |
| 执行阶段 | "决定买, 怎么买入" | 快, 直接成交, 不和价格较劲 |

最大错误: 把决策阶段的"克制"延伸到执行阶段。

### 期望值检验 (任何挂单前必算)

```
节省金额 < 错过概率 × 错过损失 → 直接市价
节省金额 > 错过概率 × 错过损失 → 可挂限价
```

典型反例: 500 股省 ¥250 (差 0.5%) vs 错过 ¥3,500 建仓收益 → 必须市价。

### 三档建仓的执行模式

| 档位 | 占缺口 | 距现价 | 执行方式 |
|------|-------|-------|---------|
| 起仓档 | 10-30% | -0.5% 到 -1% | 市价或近价限价 |
| 主仓档 | 40-50% | -3% 到 -5% | 限价挂回调位 |
| 底仓档 | 20-30% | -8% 到 -12% | 限价挂深度位 |

### "决策成熟" 4 项全过

```
✅ 标的已研究 (主线 / 评分 / 仓位规划)
✅ 仓位规划已定 (核心 / 普通 / 试错档目标 %)
✅ 价格不极端 (RSI(6) ≤ 80, 距 60 日均线 ≤ +15%)
✅ 时机不极端 (近 7 日累涨 ≤ 8%, 当日涨幅 ≤ 3%)
─────────
4 项不全 → 不是"决策成熟", 是"标的认定但时机不对"
```

### 不要犯的执行错误 (实证)

```
❌ 错误 1: 把"反 FOMO" 和"挂限价单"绑定
❌ 错误 2: 用"主仓价"当"起仓价"挂单
❌ 错误 3: 持续 30 分钟在挂单价上方还死守原单
❌ 错误 4: 起仓档限价卡到 -2% 以下 (跨入主仓档区间)
   实证 3 次 (5/7 易点 / 5/7 润泽 / 5/8 润泽差 ¥0.08):
   起仓档 -2% 限价 = 5-15% 概率"差几分钱"没成交
   损失分布: 错过单 ¥500-7150 (远高于 -2% 节省的 ¥100-300)
   修正: 起仓档限价严格 ≤ -1.5%, 优先 -0.5% 到 -1% 或市价

❌ 错误 5b: 退出标的只设"反弹清仓"单, 没设"下破止损"
   实证 (5/7-5/8 平高): playbook 写"¥23 上破清仓", 但综合分 6.6 已属"应清"
   5/7 收盘 ¥22.21 → 5/8 ¥21.38 阴跌 -3.74%, 两日额外亏 -¥4,814
   修正: 退出标的必设双触发 (上破反弹清仓 + 下破止损清仓)

❌ 错误 5: 触发位越过当日 跌停/涨停 物理边界
   实证 (5/8 兆易 ¥315): 距昨收 ¥350 跌幅 -10.04% 已超主板跌停, 当日不可能成交
   修正: 设触发位时必校核
     ① 距昨收涨跌幅 < 10% (主板) 或 < 20% (创业板/科创)
     ② 距涨跌停物理边界 ≥ 0.5% 缓冲
     ③ 距盘中价 vs 距昨收 两个锚点都要算
─────────
减仓 / 止盈也适用同样原则。"慢决策, 快执行"。
```
