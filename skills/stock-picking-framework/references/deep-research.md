# Deep Research Methodology (深度研究方法论)

> 何时加载: 做新标的研究 / 评分修正 / 季度复盘时。
> 解决问题: "看一个数字就下结论 / 反复修正" 的研究失真。
> 真实痛点: 易点研究 V4 → V9 多次修正, 都是用户校验后才发现错误。
> 2026-06-24 新增痛点: 粤电力A研究 — 看到低净利率高PE就跳过数据拉取直接下结论, 用户校验后发现Q1毛利率翻倍/亏损收窄60%被遗漏。
> 核心原则: **一次研究到位, 不要靠用户校验来发现错误**。
> 版本: V14.6 (2026-07-07) — 语义保持重构: 闸门前置 + 反前瞻措辞收敛

## 最高优先级硬闸门 (先读, 违反即重做)

> 语义保持: 本节不改变评分、仓位或买卖纪律，只把原本分散在后文的身份核验、领先指标、同行 alpha 和反前瞻规则前置。

```
任何研究价值判断 / 建仓 / 加仓 / 减仓 / 止损复核, 必须先输出:

0. target_identity_check
   - 代码 / 简称 / 主营 / 行业 / 当前市值
   - 入库前必须核对标的真实性, 不能只信 watchlist 或用户口述名称

1. next_quarter_nowcast
   - 下一季 / 当季 Bear / Base / Bull 三档收入、净利或关键变量区间
   - 不能只引用已披露旧财报

2. leading_indicators
   - 榜单 / 订单 / 价格 / 出货 / 投放 / 被投标的股价 / 同行先行数据
   - 标注: 数据期间 + 来源 + 指向哪个财务科目
   - 必须包含同行最新季 alpha 对比；若找不到可比同行, 说明原因

3. stale_data_check
   - 当前结论是否依赖过时数据?
   - 若缺少能判断下一季的领先证据, 只能输出"证据不足, 需补预研"

4. action_implication
   - 预期仓 / 确认仓 / 等证据
   - 仓位上限必须跟证据强度匹配

5. data_unit_check
   - 财报百分比、同比、环比必须标注 raw_value / display_value / 单位
   - 同花顺等汇总源若返回 0.19032 这类小数, 必须先确认是 0.19032% 还是 19.032%
```

**短句版**: 不做预研预估, 不许下结论。旧财报只作基准校验, 不能替代 next_quarter_nowcast。
**反前瞻禁句**: 不得把"等中报/等财报确认"作为行动理由。真正能等的只有"领先指标不足"; 财报是验证节点, 不是行动闸门。

## 执行闸门系统 (V14.5 新增, 防"看一眼就下结论")

> **触发案例 (2026-06-24)**: 粤电力A研究 — 看到515亿营收→6亿净利(1.2%净利率)+PE 43x, 立即判定"垃圾, 不研究"。
> 用户逼问后才拉Q1数据: 营收+7.4%, 亏损收窄60%, CFO+64%, 毛利率翻倍(4.5%→7.0%)。
> 结论可能仍是"不建仓", 但**过程错了** — 框架有内容规则但缺少执行闸门, 跳过了数据收集直接下结论。

### 闸门 1: 数据拉取清单 (研究启动前强制执行)

**任何标的, 不论第一印象好坏, 必须先核验标的身份并拉齐以下数据才能出任何结论:**

```
□ 0. 标的身份核验 (代码/简称/主营/行业/市值)
     检查: 代码和名称是否一一匹配? 行业是否与研究逻辑一致?

□ 1. 最新季报三表 (营收/归母/扣非/CFO/毛利率 + yoy/qoq)
     来源: 巨潮公告 / 新浪财报 / 腾讯行情
     检查: 是不是最新一期? (不要用旧季报)
     口径: 百分比/同比/环比必须标注 raw_value + display_value + 单位

□ 2. 最近 4-8 个单季趋势 (累计→单季还原后)
     检查: 趋势是改善还是恶化? (不能只看最新一个点)

□ 3. 估值锚 (当前市值 / TTM净利 / fwd PE / 同行对比)
     检查: 价值锚 = 自主推演净利 × 自主 PE 中枢 (R1b/R10)
           券商目标价 / 一致预期只能旁注做 sanity check, 禁止当锚
           (反例 2026-06-24: 润泽复评用"现价超券商目标价62.4"当减仓理由 = 盲信券商)

□ 4. 券商一致预期 (≥ 3 份近 12 月研报)
     检查: 市场定价的是什么情景? 我有什么不同?

□ 5. 缠论周线/日线结构 (一买/二买/三买/中枢/背驰)
     检查: 当前位置在结构中的哪一段?

□ 6. 同行最新季 alpha 对比 (至少 3 家可比公司; 不足则说明原因)
     检查: 标的是公司 alpha、行业 beta, 还是 alpha 损失?
```

**闸门逻辑**: 身份核验 + 数据拉齐 → 进入闸门 2。缺任一项 → 不能出"不研究/不建仓/建仓"任何结论, 只能输出"数据不全, 待补"。

### 闸门 2: 正反双方证据 (下负面结论前强制执行)

**任何"不研究/不建仓/否决"结论, 必须先列出 ≥ 3 条反方证据 (支持买入的理由), 且每条必须基于已拉取的数据。**

```
正面证据 (支持买入):
  1. <基于数据的正面发现>
  2. <基于数据的正面发现>
  3. <基于数据的正面发现>

反面证据 (支持否决):
  1. <否决理由, 必须引用具体数据>
  2. <否决理由, 必须引用具体数据>
  3. <否决理由, 必须引用具体数据>

综合判断: 正面权重 vs 反面权重 → 结论
```

**闸门逻辑**: 如果找不出 ≥ 3 条正面证据 → 说明数据拉取不完整, 回到闸门 1。只有正反双方都充分论证后, 才能下结论。

### 闸门 3: "不研究"也需要完整流程

**"不研究"是一个研究结论, 不是一个跳过研究的借口。**

```
❌ 错误路径:
  第一印象差 → "垃圾, 不研究" → 跳过所有流程 → 结束

✅ 正确路径:
  第一印象差 → 闸门 1 (身份核验 + 数据拉齐) → 闸门 2 (正反双方证据)
  → 基于数据判断: 是真的差还是第一印象错了?
  → 结论 (基于完整数据):
      a) "经完整流程, 确认不建仓, 原因: ..."  ← 这是有效结论
      b) "数据揭示了第一印象未见的改善, 值得深度研究"  ← 纠正偏见
      c) "证据不足, 先放远档观察, 等领先指标补齐"  ← 诚实悬置；财报仅作验证节点
```

**关键纪律**: 花 3 分钟拉数据可能发现被忽略的改善信号 (粤电力A: 毛利率翻倍、亏损收窄 60%)。即使最终不建仓, 这个发现过程本身就是研究的价值。

### 强制执行序列 (8 步, 不可跳步)

```
Step 0: 读入闸门系统 (本段) — 确认理解"不能跳步"
Step 1: 完成标的身份核验 + 拉齐闸门 1 数据
Step 2: 列正反双方证据 (闸门 2, ≥ 3+3)
Step 3: 判定是否值得深度研究 (基于数据, 不是第一印象)
Step 4: 如需深度研究, 进入八维度框架 (维度 0-7)
Step 5: 出评分 + 三档情景
Step 6: 缠论结构分析 (买卖点 + 止损)
Step 7: 仓位决策 + 操作建议
Step 8: 写入研究文件 + 更新 watchlist/playbook

违反任一 Step → 研究作废, 重做。
```

**闸门违规自检**: 研究过程中每 5 分钟问自己一次: "我现在下的结论, 是基于完整数据还是基于第一印象?"

#### 4 个 bottom-up 核心问题 (必答)

1. **产能/装机的硬数据**: 当前装机 X, 在建 Y, 何时投产? (举证: 公告原文 + 研报)

### 维度 1: 业绩三层利润 (不能只看一层)

```
表层: 归母净利 (法定披露, 包含所有损益)
中层: 扣非净利 (剔除非经常性损益)
深层: 剔除特殊项后扣非 (剔除汇兑+股份支付+投资收益)
─────────
三层全列, 才能识别"主营 vs 特殊项"
```

#### 维度 1.1: 累计 → 单季还原 (强制, 2026-05-31 新增, 防"易点失速"误判)

> **痛点 (真实失误)**: akshare/巨潮的利润表与营收是**累计 YTD 口径** (年初至今, 非单季)。
> 直接拿来算 TTM / 同比 / 环比 / 增速形容词, 会系统性出错。
> 实证 1: 百济 26Q1 "环比" 若用累计算 = +10%, 单季还原后 = **+399%** (差 40 倍)。
> 实证 2: 易点 营收凭印象写"+40%→+12%腰斩", 单季还原后 = 25Q1→26Q1 仅 9.29→10.42 (+12%, **从无 +40%, 更无腰斩**), 是高基数回落。

**强制流程 (任何"增速/环比/趋势"结论前必做)**:

```
Step 1: 把累计 YTD 还原成单季 (公式)
  单季 Qn = 累计(Qn) - 累计(Qn-1)
  例: 25Q4单季 = 全年 - Q1-Q3累计;  25Q2单季 = H1 - Q1
  (Q1 本身即单季, 无需还原)

Step 2: 列出连续 ≥ 5 个单季数列 (营收 + 归母 + 扣非)
  25Q1 / 25Q2 / 25Q3 / 25Q4 / 26Q1 ...

Step 3: 所有增速基于"单季对单季"
  yoy = 26Q1单季 / 25Q1单季 - 1   (同季比, 避开季节性)
  qoq = 26Q1单季 / 25Q4单季 - 1   (环比)
  TTM = 最近4个单季之和 (≠ 任一累计值)

Step 4: 增速形容词举证 (零印象)
  "失速/腰斩/加速/断崖" 等词 → 必须先贴出上面的单季数列, 数列支持才能用
  禁止: 不列单季数列就写"+X%→+Y%腰斩"
```

**校验清单 (评分前必过)**:
```
□ 列出了 ≥ 5 个单季数列 (不是只有累计值)?
□ TTM = 4个单季之和 (不是直接拿某个累计值)?
□ yoy 是"同季比" (26Q1 vs 25Q1), 不是"累计比"?
□ 任何"失速/腰斩/加速"形容词, 都有单季数列支撑?
□ 高基数效应已区分 (增速回落 ≠ 失速, 看绝对值是否还在增长)?
```

**建议自动化**: R0 的 `fetch_stock_data.py` 输出已是累计值, 研究时手算单季易错。
→ 后续可给脚本加 `--single-quarter` 选项自动还原单季 + 算 yoy/qoq/TTM, 从根上消除人为口径错误。

### 维度 2: 异常项根因追问 (5 个为什么)

```
表面: 净利转亏 / 暴增
  ↓ 为什么?
第 1 层: 看公告"非经常损益"明细
  ↓ 是什么?
第 2 层: 公允价值变动 / 投资收益 / 减值损失
  ↓ 涉及什么标的?
第 3 层: 战略持仓 / 联营企业 / 金融资产
  ↓ 标的走势?
第 4 层: 二级市场价格变化 / 上市事件
  ↓ 持续性?
第 5 层: Q2/Q3 修复 / 恶化预期
─────────
5 层不到根因, 不出研究结论

任何指标偏离 ±30% 必须追问:
  ✅ 营收 +50% / -30% → 业务变化原因?
  ✅ 净利 +100% / -50% → 一次性还是持续?
  ✅ 扣非 vs 归母差异大 → 非经常项明细?
  ✅ 公允价值变动 → 持仓什么?
  ✅ 投资收益激增/巨亏 → 哪个被投?
```

#### 维度 2.5: 会计幻觉过滤器 (2026-06-25 精简)

> **决策只用这一句原则**: **剔除一次性损益(公允价值/汇兑/投资收益/减值/补助)看真实主营, 同一笔不在维度 2/3/6/7 重复扣分。** 一次性占比小(<50% 表层归母)时直接用表层, 大时才剥离。
> ⚠️ 原"定档表 + 75% 摩擦系数 + 逐维度加减档"僵硬机制已砍(易点 1 例, n=1 过拟合), 存档 `cases/archived-rules-2026-06-25.md`。**下面的详细机制降级为备查, 决策不用, 只用上面原则 + 判断。**
> 实战检验(2026-06): 江波龙 26Q1 38.6亿——非经常仅~1.4%→不剥离, 是真实涨价主营(反向陷阱); 美的 26Q1 表观+2% 但剔汇兑高基数后核心经营+10.8%。两例都靠这一句原则, 没用那套定档表。

**(以下为备查机制, 非决策必跑)** 核心问题: 5 个为什么追到根因后, 评分体系没有自动用"调整后利润"替换"表层归母", 导致一次性事件污染整个评分.

**强制流程 (维度 2 完成后必走)**:

```
Step 1: 列出所有"特殊项" (一次性损益)
  □ 公允价值变动 (持有金融资产 / 战略股权)
  □ 投资收益 (大额对联营 / 处置)
  □ 资产减值 / 信用减值
  □ 汇兑损益 (海外业务汇率波动)
  □ 股份支付 (高管激励)
  □ 政府补助 / 税收返还
  □ 资产处置利得 / 营业外收支

Step 2: 每个特殊项判定"持续性"
  
  | 性质 | 判定依据 | 评分对冲 |
  |------|---------|---------|
  | 一次性 (会计幻觉) | 来源是市值波动 / 事件 / 预期外 | 用调整后利润评分 |
  | 经常性 (持续) | 来源是常规对联营投资收益 / 政府常规补助 | 维持表层利润评分 |
  | 模糊 (待验证) | 既可能反复又可能停 | 用 50% 权重 |

Step 3: 三层利润 → 评分用"最深层"
  
  深层 = 表层归母 - 一次性损益 + 一次性损益的 25% 摩擦成本
        (摩擦成本: 即使一次性也会有税/利息影响, 留 25% 折扣)
  
  实证 (易点 26Q1):
    表层归母 = -0.6354 亿 (亏损)
    扣非 = +0.1604 亿
    APP 公允价值变动 = -1.385 亿 (一次性, 已识别根因)
    
    深层 = -0.6354 + 1.385 × 75% (摩擦折扣) = -0.6354 + 1.0388 = +0.4034 亿
    或更严谨的"剔除汇兑+股份支付"算法 = +0.68 亿
    
    评分用深层 +0.68 亿, 不用表层 -0.6354 亿

Step 4: 评分对冲规则 (强制)
  
  若 一次性损益占比 (|一次性| / |表层归母|) > 50%:
    必须 走对冲规则:
    
    维度 2 (基本面底座): 用深层利润评分 (不用表层)
      - 深层 yoy > +10% → 维度 2 ≥ 7.0
      - 深层 yoy > +20% → 维度 2 ≥ 7.5
    
    维度 3 (预期差): 一次性反转就是预期差 +0.5-1.0 分
      - 若 一次性方向是负 (拖累表层) + Q+1 大概率反转 → +1.0 分
      - 若 一次性方向是正 (虚高表层) + 不可重复 → -0.5 分
    
    维度 5 (催化集群): Q+1 反转就是催化 +0.5-1.0 分
      - 若 已识别 反转触发 (APP 反弹 / 减值冲回 / 处置完成) → +1.0 分
    
    维度 6 (估值): 用深层 TTM 重算 PE, 不用表层 TTM
      - 表层 TTM 含一次性, 算出 PE 失真
      - 深层 TTM = 表层 TTM 剔除一次性后的 TTM 利润
    
    维度 7 (持续性): 看主营趋势, 不被一次性影响
      - 主营 (深层) 仍正增长 → 维度 7 维持评分

  ❌ 严禁 (一次性事件不允许重复扣分):
    维度 2 按表层 -1 (基本面差) 
    维度 3 按表层 -1 (无预期差)
    维度 6 按表层 -1 (估值贵)
    维度 7 按表层 -1 (持续性弱)
    → 同一事件被扣 4 次, 评分系统失效

Step 5: 评分稿出来后回查 (校验清单)
  □ 维度 2 用了深层利润? (不是表层)
  □ 维度 3 是否给"一次性反转 = 预期差"加分?
  □ 维度 5 是否给"Q+1 反转 = 催化"加分?
  □ 维度 6 PE 是否用深层 TTM 重算?
  □ 没有同一事件被多个维度重复扣分?
```

**实证对比 (易点 26Q1)**:

```
错误评分 (V1, 不用过滤器):
  维度 2: 5.0 (表层亏损)         ← 重复扣 -1
  维度 3: 5.5 (没看到反转预期差)   ← 重复扣 -0.5
  维度 6: 4.5 (TTM PE 647x 失真)   ← 重复扣 -1
  维度 7: 5.5 (主营 +12% 偏低估)   ← 重复扣 -0.5
  综合: 6.35 (失真)

正确评分 (V1.2, 走过滤器):
  维度 2: 7.0 (深层 +12.45%, 主营稳健)
  维度 3: 7.5 (Q2 APP 冲回 +1.0-1.4 亿是高确定性预期差)
  维度 5: 8.0 (Q2 中报 + APP Q2 财报 + NVDA GTC 多重催化)

```

**何时不走过滤器** (一次性占比 < 50% 或经常性):
- 一次性占比 < 50%: 直接用表层评分
- 经常性 (持续): 直接用表层评分 (这就是真实业务结果)
- 模糊待验证: 走 50% 权重 (维度 2 用半折深层)

#### V14.3 新增: 同行最新季 alpha 判定 (关键 Bull/Bear 调整证据)

> **核心洞察**: 公司 alpha vs 行业 beta 是决定 Bull 概率上调幅度的硬证据
> **真实案例**: 润泽 2026 Q1 营收 +53.55% vs 光环 -10.86% / 数据港 -3.80% → 行业整体下滑下润泽严重 outperform → Bull 概率从 30% 直接上调到 45%

#### 操作 SOP

```
Step 1: 列出 3-5 家同行业最直接可比公司 (业务相似度 > 60%)

Step 2: 拉取最新一季营收/净利同比 (`a-stock-data` 新浪财报三表 + 巨潮公告；akshare 仅作 fallback)

Step 3: 横向对比表 (必出)

| 标的 | 最新季营收同比 | 行业地位 | alpha/beta 判定 |
|------|--------------|---------|----------------|
| 标的 X | +X% | 龙头 | ? |
| 同行 A | +Y% | 老二 | ? |
| 同行 B | +Z% | 老三 | ? |
| 同行 C | -W% | 边缘 | ? |
| 行业均值 | M% |        |        |

Step 4: 三种结论 (按差距大小)

  情景 1: 标的同比 - 行业均值 > +20pp (强 alpha)
    → Bull 概率 +10pp (最高 +15pp)
    → 解释: 公司有结构性优势 (产品/客户/技术)
    例: 润泽 +53.55% vs 行业 -7% = +60pp 差距 → Bull +15pp
    
  情景 2: 标的同比 ≈ 行业均值 ± 10pp (行业 beta)
    → Bull 概率不调整, 关键看行业景气度
    → 解释: 没有公司 alpha, 跟行业波动
    
  情景 3: 标的同比 - 行业均值 < -10pp (alpha 损失)
    → Bear 概率 +10pp
    → 解释: 公司丢市占率 / 客户流失 / 产品迭代落后
    例: 美的 Q1 +2.55% vs 同行格力/海尔 +5-8% → Bear +10pp

Step 5: 必须保留同行 Q1 数据快照在研究文件 (用于后续验证)
```

#### 同行 Q1 alpha 判定的 3 个常见误用

```
🚫 误用 1: 只对比 1 家同行 → 样本不足
   ✅ 正确: 至少 3 家, 包含龙头/老二/老三

🚫 误用 2: 用全年数据对比 → 季度 alpha 看不出
   ✅ 正确: 对比同期 (Q1 vs Q1, 不能 Q1 vs 全年)

🚫 误用 3: 不区分细分赛道 → 算力服务器 vs 传统 IDC 不可比
   ✅ 正确: 业务结构类似才比 (润泽 vs 光环可比, 但 vs 工业富联不可比)
```

### 维度 6: 增速可持续性 (高增速标的必查)

```
触发条件: 任一指标 yoy ≥ +30% 或 ≤ -30%, 必须做可持续性分析
解决痛点: "Q1 +94% 是加速起点还是基数顶点?" 这种关键问题不能拍脑袋

7 步分析法 (必须全覆盖):

Step 1: 基数效应推演 (季度同比拆解)
  → 列出标的过去 4-8 个季度的营收/净利基数
  → 推演未来 4-8 个季度同比 (Q1/Q2/Q3/Q4 + 下一年)
  → 判断: 是"高速→中高速健康减速" 还是"断崖失速"
  
  典型节奏 (健康):
    +56% (Q1) → +50% (Q2) → +40% (Q3) → +30% (Q4) → +27% (次年)
  典型节奏 (危险):
    +56% (Q1) → +30% (Q2) → +5% (Q3) → -10% (Q4) ⚠️ 断崖

Step 2: 量价拆解 (利润可持续性核心)
  营收增长 = 量增 × 价升 (or 价降)
  → 量贡献 vs 价贡献分别多少?
  → 量是结构性 (新产品周期) 还是周期性 (一次性补库)?
  → 价是 ASP 自然提升 (产品升级) 还是涨价红利 (周期高位)?
  
  毛利率分析:
    净利增速 > 营收增速 → 毛利率/费用率改善, 看持续性
    净利增速 < 营收增速 → 价格战 / 投入期, 警惕

Step 3: 订单可见度 (硬证据)
  6 个月可见度: 已锁单 / 在手订单 / 客户指引
  12 个月可见度: 框架协议 / NRE 早期订单
  24 个月可见度: 下一代产品/平台储备
  → 用百分比量化: 6 月 95% / 12 月 80% / 24 月 60%

Step 4: 客户/下游 Capex 节奏 (最大变量)
  → 列出 Top 3-5 客户的 Capex 指引 (来自客户财报)
  → 客户集中度风险: Top 5 占比 > 50% 必须重点关注
  → 关键观测信号 (按时点排序): 哪些财报会验证可持续性

Step 5: 竞争格局 (份额风险)
  → 同业份额变化趋势 (扩张 / 持平 / 压缩)
  → 下一代产品/平台是否会重新分单
  → 价格战可能性

Step 6: 三档情景假设 (Bear/Base/Bull) + 概率加权
  ┌──────┬──────┬──────┬──────┐
  │ 情景 │ 营收 │ 净利 │ 概率 │
  ├──────┼──────┼──────┼──────┤
  │ Bear │ ?%   │ ?%   │ ?%   │
  │ Base │ ?%   │ ?%   │ ?%   │
  │ Bull │ ?%   │ ?%   │ ?%   │
  └──────┴──────┴──────┴──────┘
  期望值 = Σ(情景 × 概率)

Step 7: 可持续性证伪信号 (硬触发)
  列出 4-6 个具体触发条件, 任一触发→评分下修 1-2 分
  例:
    🚨 客户 Capex 指引下修 ≥ 10%
    🚨 标的中报增速跌破阈值 (反转 ≥ 15pp)
    🚨 政策风险升级 (关税/出口管制)
    🚨 同业份额超预期挤压
    🚨 毛利率连续两季下滑 ≥ 1pp
─────────
输出: 短期 (3-6 月) / 中期 (6-18 月) / 长期 (18-36 月) 三档评级
      + 一句话核心判断 ("骑乘 X 个季度兑现期" vs "押注永续高增长")
      + 可持续性评级 ⭐ (1-5 星)
```

### 维度 6.1: 前瞻验证 — 不等财报, 用领先指标判断下一季 (P1 + 最新资料优先级 的用法澄清, 非新增 R)

> **触发更正 (2026-06-03)**: AI 用"反转仅 1 季未验证 → 等中报盖章"当作百济升档闸门, 被用户驳回。
> 这违反 SKILL.md「最新资料优先级」——**已披露财报是优先级最低的一档**。等中报 = 等最慢、最滞后的信号,
> 利好(放量/催化)那时早已 price in, 预期差吃不到, 只剩接盘。**做预期差的本质 = 比财报更早判断下一季。**

#### 闸门移位 (核心)

```
错(被动滞后): N 季反转 → 等下一份财报盖章 → 再动        [利好已落地, 没预期差]
错(另一极端): 评分高 + 反转了 → 直接买                  [把行业β当公司α, FOMO]
对(前瞻验证): N 季反转 → 主动挖领先指标 → 对"下一季能否延续"做三档概率
    ├─ 领先指标够硬 + 赔率好 → 预期差就在【现在】, 现在动 (按风险维度定档/定量)
    └─ 领先指标挖不到 / 互相矛盾 → "等"; 但等的是【证据不足】, 不是【日历到期】
```

**关键纪律**: 真正该"等"的理由只有一个 —— **我还没拿到能判断下一季的领先证据**。
"财报还没出" 不是理由。拿得到前瞻证据就现在动, 拿不到才等。
**这不是"评分高就买"的许可证**: 前瞻验证是更难的活 (要真产出领先证据), 不是更松的活。

#### 强制输出 (2026-06-12 升级)

任何研究价值判断 / 建仓 / 加仓 / 减仓 / 止损复核, 必须输出 `next_quarter_nowcast`:

```
next_quarter_nowcast:
  period: 2026Q2
  stale_data_check: 当前结论是否只依赖旧财报? 若是 → 研究无效
  leading_indicators:
    - 指标名称 / 数据期间 / 来源 / 指向的财务科目 / 置信度
  bear_base_bull:
    Bear: 收入/净利/关键变量区间 + 触发条件
    Base: 收入/净利/关键变量区间 + 触发条件
    Bull: 收入/净利/关键变量区间 + 触发条件
  expectation_gap:
    市场当前担心什么? 领先指标是否证伪? 何时被重估?
  action_implication:
    预期仓 / 确认仓 / 等证据, 以及仓位上限
```

缺任一项 → 只能输出"证据不足, 需补预研", 不允许给出建仓/清仓结论。

#### 游戏股 / 出海股反推模板 (三七案例沉淀, 非新增 R)

```
Step 1: 锁定核心产品
  新品 / 存量大 DAU 产品 / 海外增长产品分别列出。

Step 2: 用榜单反推月流水
  - Sensor Tower / AppMagic 给出的 IAP 估算优先
  - 若只有排名: 用同榜相邻产品公开流水区间反推
```

#### 下一季延续性 — 三档概率模板 (P2)

```
对"下一季反转是否延续", 而非"等它发生", 显式给三档:
  延续(高/中/低) : 领先指标几条共振指向继续改善?
  停滞(高/中/低) : 一次性因素退潮 / 高基数 / 价格见顶?
  逆转(高/中/低) : 出现左侧信号反向 (价格再跌/订单转弱/汇兑反噬)?
→ 延续概率高 + 赔率(向上空间/向下空间)好 → 预期差在现在, 按风险维度定档建仓
→ 三档说不清 (领先指标缺失/矛盾) → 这才是"等"的正当理由
```

#### 反转股尤其适用 (呼应 SKILL.md "反转股别用错尺子")

```
周期/困境反转股早期天然: 当期数据难看 + 只有 1 个改善季。
拿"等下一份财报"做闸门 = 必然错过左侧最肥的一段 (实证: 华灿 ¥7 时左侧 4/5 命中却被打低分观望)。
反转股的前瞻验证 = 左侧信号计数 (扣非环比收窄/毛利率转正/价格止跌/现金流转正/第二曲线)
                  + 龙头是否先行修复 + 上述领先指标。够 → 现在左侧布局 (按试错/普通档定量)。
```

> **结构性反转 vs 周期性反转 的延续性权重不同**:
> 结构性 (全球放量/经营杠杆/国产替代爬坡) → 单季延续概率天然高, 不易一季打脸。
> 周期性 (价格反弹/补库) → 单季可能是 head-fake, 需更多左侧信号 + 龙头先行确认。

### 维度 7: 前瞻性催化集群

```
关键洞察: "前瞻性" 不是看单点催化, 是看"催化集群"

催化分类:
  A. 行业催化集群 (多个新技术/产品同时发生)
  B. 政策催化集群 (政策叠加效应)
  C. 上下游联动 (产业链同步发酵)
  D. 海外映射 (美股/港股领先 → A 股跟进)

评估方法:
  单点催化 影响: +5-10% (估值修复)
  集群催化 影响: +30-100% (估值重估)
  生态催化 影响: 商业模式重塑 (PE 锚改变)

判断催化等级的 4 问:
  1. 是单点还是集群? (越多催化叠加越强)
  2. 影响哪个财务指标? (毛利率/营收/费用)
  3. 量级和时点? (Q2 还是 Q3, +5pp 还是 +20pp)
  4. 是否已 price in? (市场反应 vs 真实空间)

详见 catalyst-cluster.md
```

## 数据查询优先级

```
关键数据必须查公告原文, 不依赖搜索摘要

优先级 (从高到低):
  1️⃣ 巨潮资讯网 / 上交深交所披露 (公告原文)
  2️⃣ 公司投资者关系活动记录 / 业绩说明会
  3️⃣ 主流财经媒体深度报道 (财新 / 界面)
  4️⃣ 券商研报 (注意发布日期 + 评级)
  5️⃣ 财经网站汇总数据 (同花顺 / 东方财富, 时延 1-3 个月)
  6️⃣ 搜索摘要 (仅用于初步定位, 不用于结论)

特别警惕:
  - 摘要里的"扣非 +12%" 可能是"剔除特殊项后" 的版本
  - 摘要里的 PE 可能是几个月前的过时数据
  - 摘要里的"营收增长" 可能掩盖"利润下滑"
```

```

  ❌ 直接抄财经网站的 PE / PB
  ✅ 当前市值 ÷ 估算净利 = 当前 PE

规范 3: 关键数字必须二次校验

  - 引用的财报期间是哪个?
  - 与最新公告是否一致?
─────────
```


## 财报硬性查证规则（最高优先级，违反即重做）

```
铁律 1: 必查"当前最新一期" 已披露的报告
  ❌ 错误: 只看年报 +14% 就出结论 (美的 V1 失误)
  ✅ 正确: 研究开始前先问 "当前最新一期已披露的是哪个? 我用的是它吗?"
  
  执行流程 (研究第一步):
    □ Step 1: 查公司公告列表 / 巨潮资讯 / 投资者关系
    □ Step 2: 锁定"最新一期" (季报 / 快报 / 预告 / 中报 / 年报)
    □ Step 3: 必须用最新一期, 不能用更早的
    □ Step 4: 同时保留前一期 (用于环比) + 去年同期 (用于同比)
  
  披露日历参考 (典型情况, 以实际公告为准):
    - 1 月底-2 月初: 业绩快报 / 预告
    - 4 月底:        年报 + Q1 季报
    - 8 月底-9 月初: 中报
    - 10 月底:       Q3 季报
  
  特别提醒: 不要假设"今天是 5/7 所以 Q1 应该已发"
            必须实际查证 — 有些公司可能延迟披露
─────────
违反代价: 用旧数据指导新决策, 评分高估 1-2 分

铁律 2: 三层利润必须最新季 + 同比 + 环比
  必须列出:
    ✅ 最新季报: 营收/归母/扣非 + 同比 + 环比
    ✅ 上一季: 同样三项 (用于环比)
    ✅ 去年同期: 同样三项 (用于同比)
    ✅ 全年(若有): 同样三项 (用于趋势)
  
  特别警惕"反转信号":
    若年报 +X% 但最新季报 -X% (反转 ≥ 20pp)
    → 优先按"反转季"评估, 不按年报

铁律 3: "利空出尽 vs 业绩证伪" 必须双校验
  关键问题: 营收增速是否仍在 +10% 以上?
  
  ✅ 营收 ≥ +10% + 扣非短期负 = 投入期 / 利空出尽
     (例: 汇川 Q1 营收 +13% → 评分 7.10 合理)
  ❌ 营收 < +5% + 扣非负 = 失速期 / 业绩证伪
     (例: 美的 Q1 营收 +3% → 评分降至 6.30)
  ⚠️ 营收 5-10% + 扣非负 = 灰色区, 看其他指标
─────────
违反代价: 把"业绩证伪"误判为"利空出尽" → 错误加仓

铁律 4: 同行业横向校验
  当某公司"看起来利空出尽" 时, 必须问:
    □ 同行业是否同步业绩压制? (是 = 行业共性)
    □ 行业共性的话, 哪家是真投入期 vs 真失速期?
    □ 龙头是否先行修复? (周期反转标准)
─────────
违反代价: 把行业 β 当公司 α (例: 把美的当汇川)
```

## 真实失误案例

```
案例: 2026/5/7 美的研究 V1 → V2 修正

V1 失误:
  只看 2025 年报 +14%, 没查 2026 Q1
  评分 7.55, 建议 8-10% 普通档建仓
  
V2 真相 (用户提醒后查证):
  Q1 2026 扣非 -14% (反转 29pp)
  营收仅 +2.55% (失速)
  评分修正 6.30, 不达建仓门槛

教训:
  □ 任何 5 月研究必须查 Q1 (4 月底已披露)
  □ 收购标的亏损 ≠ "一次性" (持续拖累)
  □ 不能"看起来 PE 13x 就建仓" 
  □ 营收增速是 "投入期 vs 失速期" 的核心分水岭

沉淀位置: cases/2026-05-07-research-failures.md
```
