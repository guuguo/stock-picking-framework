# Quantitative Triggers (量化触发规则)

> 何时加载: 准备建仓/加仓/减仓时, 或每日盘前盘中扫描时。
> 配套脚本: `scripts/quant_scan.py`

## 八维量化指标

| 维度 | 指标 | 健康区间 | 极端区间 |
|------|-----|---------|---------|
| **趋势** | 距 60 日均线 | -5% ~ +10% | > +20% (透支) / < -20% (低位) |
| **短期动能** | RSI(6) | 35-70 | > 80 极端超买 / < 20 极端超卖 |
| **中期动能** | RSI(14) | 40-65 | > 75 偏热 / < 30 偏冷 |
| **加速度** | 14 日累涨幅 | < +10% | > +20% 加速透支 |
| **量价** | 量比 (5日/20日均量) | 0.8-1.5 | > 2 巨量 / < 0.7 缩量 |
| **估值** | PE 历史百分位 | < 70% | > 90% 顶部 |
| **业绩** | Q1 净利同比 | > +20% 优秀 | > +100% 爆发 |
| **主线** | 主线分级 | β / α / β+α | 非主线 减仓优先 |

## 综合打分 (2026-06-25 精简: 砍固定权重)

> **8 维(上表)= 研究覆盖清单**(确保不漏维度), **不加权算综合分**。
> 综合分用 **0.5 档主观判断**(框架去伪精确口径): **≥8 核心档 / 7-8 普通 / 6-7 试错 / <6 不建/减**。

## Kelly 公式（半 Kelly）

```
最优仓位 f = (p × b - q) / b / 2

p = 上涨概率 (主观估算)
q = 1 - p
b = 上涨幅度 / 下跌幅度

挂单 / 建仓前必算:
  f ≤ 0     → 期望值为负, 不建仓
  f 0-5%    → 试错档
  f 5-10%   → 普通档
  f > 10%   → 核心档 (但仍受 20% 上限约束)
```

## 起仓档量化通过线（4 项全过）

```
✅ 综合分 ≥ 7.0
✅ 距 60 日均线 ≤ +15%
✅ RSI(6) ≤ 75 且 14 日累涨 ≤ +15%
✅ Kelly f > 0
─────────
任一不过 → 不是起仓时机
4 项全过 → 起仓档执行 (按 SKILL.md Execution Discipline)
```

> 起仓档(试用仓 ≤2%) 允许左侧试探，周线主跌浪不阻断起仓。主跌浪限制见下方升档条件。

## 主仓/加仓升档条件 (V14.4 新增)

> **动机**: 三七(营收-12%扛在主跌浪, 评分8.05→止损¥34k) / 平高(12.5%仓位扛在主跌浪, 止损¥33k)。
> 小仓试探可以，重仓必须等右侧。亏损不是"买了跌的票"，而是"在主跌浪里打满仓"。

```
4 项全过 → 允许起仓档 (试用仓 ≤2%, 可左侧试探)
4 项全过 + ✅ 周线趋势非主跌浪 → 允许升至主仓档 / 加仓档
```

### 周线趋势非主跌浪 — 判定方法

```
判定方法 (周线K线, 近12周):
  主跌浪 = 周线高点下移 + 周线低点同时下移 (4项满足2项的周线版)
  
  步骤:
  1. 拉取近12周K线, 识别swing高点(局部最高)和swing低点(局部最低)
  2. 高点序列: 最近2个swing高点, 后者 < 前者 → 高点下移
  3. 低点序列: 最近2个swing低点, 后者 < 前者 → 低点同时下移
  4. 高点下移 + 低点同时下移 → 认定主跌浪

主跌浪期间:
  - 起仓档(试用仓 ≤2%): 允许, 左侧试探
  - 主仓档 / 加仓档: 禁止升档, 已建主仓评估是否降回试用仓
  - 解除条件: 周线收盘站回MA5周线 + 连续2周低点抬高

例外 (需同时满足3项, 门槛严格):
  ✅ 主线硬度: 最近1季营收同比 ≥ 0% 且 扣非同比 ≥ 0%
  ✅ 均线条件: 周线收盘仍在MA10周线之上 (允许短期回调, 不允许中期破位)
  ✅ 大盘条件: 三大指数至少2个在60日均线之上 (🟢或🟡, 不允许🔴)
  
  3/3满足 → 允许升至普通档 (≤8%), 核心档仍需等右侧确认
  任一不满足 → 仅限试用仓, 不得升档

实证对照:
  三七(05-07): 营收-12% → 例外条件1不满足 → 不应升档 ✅
  平高(05月):  均线空头排列 → 例外条件2不满足 → 不应升档 ✅
  天赐(反转票): 营收连续负增长 → 例外条件1不满足 → 但天赐是反转票,
    适用P4期望贡献框架而非成长股准入, 反转票的主跌浪检查放宽为
    "等右侧确认(站回MA5周+2周低点抬高)后再加仓", 而非完全禁止。
```

## 减仓档量化触发线（满足 1 项即可减）

```
触发 1: RSI(6) > 80 + 距 60 日均线 > +20%
触发 2: 14 日累涨 > +25% + 综合分 < 7
触发 3: 当前仓位超过目标仓位 +5% 以上
触发 4: 主线被证伪 / 业绩持续 -20%+
```

## 主线偏离诊断（每周日跑一次）

```
偏离 ≥ +5% → 该主线超配, 优先减
偏离 ≤ -5% → 该主线低配, 优先补
─────────
主线分类 (用于 30% 上限计算):
  主线 1 AI 算力硬件 (光模块/服务器/芯片/存储)
  主线 2 AI 应用 + 边缘 (Agent/出海/MCU)
  主线 3 工业自动化 / 机器人 (独立主线)
  主线 4 AI 电力 (国网/配电)
  主线 5 AIDC 数据中心
  非主线: 单独管理, 总和 ≤ 15%
```


## 量化战胜直觉的真实案例

```
案例 1: 兆易创新 ¥350 (5/7)
  直觉: "决策成熟, 起仓"
  量化: RSI(6) 100 + 距 60MA +21.9% + Kelly -1.45
  结果: 撤回起仓单, 避免 ¥34k 潜在追高损失

案例 2: 工业富联 ¥64 (5/7)
  直觉: "新加观察池, 不该立刻买"
  量化: 综合分 8.6 + RSI(6) 45.5 + Kelly 0.51
  结果: 起仓 100 股, 把握下一波算力主线机会

案例 3: 隆基绿能 ¥16.53 (5/7)
  直觉: "跌得太惨, 应该止损"
  量化: RSI(6) 8.7 + 14 日累跌 -5.8%

```

## 教训

- 直觉容易被"早上的失误" / "媒体噪音" / "看到涨停" 干扰
- 量化数据是冷的, 不会被情绪带偏
- 凡是规则模糊或自我怀疑时, 立即跑量化扫描
