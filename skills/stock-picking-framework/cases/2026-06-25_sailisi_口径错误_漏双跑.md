# 案例: 赛力斯口径错误 + 漏派 dsclaude 双跑 (2026-06-25)

## 事实
- V3 (6/12) 研究赛力斯, 把**含少数股东的净利润 6.11 亿**当**归母**做同比, 得出 "26Q1 利润 -17.5%" 的结论, 据此把基本面底座维度打到 5.5。
- V4 (6/25) 拉新浪利润表核对: 26Q1 **归母 7.54 亿** vs 25Q1 归母 7.48 亿 = **+0.8% 持平**。Q1 没那么糟, 基本面底座修正回 6.0。
- V3、V4 全程纯 claude 单跑, 未派 dsclaude/codex 双跑 —— 而这台机器上两者都装了。

## 根因 (两层)
1. **口径层**: 累计→单季还原 + 归母/净利/扣非 切换, 是最容易静默出错的计算; 单跑无第二票兜底, 错误从 V3 活到 V4 才被偶然抓到。
2. **流程层 (更深)**: "双跑判断块"的建议埋在按需加载的 `model-routing.md`; 而要加载它得先决定用多模型 —— 死循环。常驻 SKILL.md 顶部把模型路由写成"默认不启用/按需/退化纯claude不报错", 没牙齿 → 时间压力下必被省。对比 `next_quarter_nowcast` 等硬闸门 ("否则结论默认无效") 每次都被执行。

## 处置 (工具层, 不计入 R 编号; 由用户直接指令 + model-routing.md 声明与冻结正交)
- SKILL.md 新增**口径双校验硬闸门**: 累计→单季还原 / 口径切换且驱动评分仓位变化的块, 必须二次校验 (有多模型→dsclaude 重算; 无→显式三表重算+标注口径); 口径未标/两跑不一致未解释→数字无效。
- SKILL.md 触发地图: 深研开篇先 `model_router.py status` 探测, 把探测器从"按需"提到"深研默认"。
- 不改成全量强制双跑: 对"主跌浪/不建仓"这类不依赖精确数字的结论, 双跑边际收益低 (governance 反过拟合)。

## 可迁移教训
- 凡看到"累计减出单季"或"净利同比", 先问一句: **归母还是净利? 标了吗?** 这是 A 股财报第一坑。
- 框架想让某动作"每次都做", 必须写成有牙齿的硬闸门 + 触发器放常驻文档; 埋在按需 reference 里 = 等于没写。
