# Stock Picking Framework

## STOCK_ROOT 发现逻辑

每次对话开始时，确定 STOCK_ROOT（所有文件操作的基准目录）：

```
1. 当前目录含 monitor/holdings.json 或 daily_playbook.md → STOCK_ROOT = 当前目录
2. 读取 ~/.config/stock-picking/config.json 的 stock_root 字段
3. 均不存在 → 询问用户两个选项：
   - 使用当前目录作为 STOCK_ROOT
   - 创建 ~/.stock/ 作为全局 STOCK_ROOT
   用户选择后自动写入 ~/.config/stock-picking/config.json，后续不再询问。

所有文件操作以 STOCK_ROOT 为基准，不写死绝对路径。
```

首次安装请运行 `bash init.sh` 完成初始化。

## Overview

A 股价值投资选股框架。核心不是"只看过去财报"，而是把 `中国未来 / 新质生产力主线 + 基本面底座 + 预期差 + 最新资料 + 催化时间` 放在一起判断。

默认偏好：

- 优先理解市场正在交易、也真正代表中国未来产业升级方向的主线
- 主线核心是 `新质生产力`（AI / 智能驾驶 / 机器人 / 边缘智能 / 算力 / 半导体 / 工业自动化 / 高端制造 / 电网升级 / 新能源核心）
- 优先真实受益、已落地兑现的公司
- 接受 `周期反转` 机会（光伏 / 锂电 / 猪周期 等供需可能改善的方向）
- 不做纯情绪反弹和纯概念博弈

## When to Use

- 分析持仓或候选股，判断该加、该减、该等
- 找"市场预期偏低，但实际可能边际转好"的标的
- 把财报、业绩预告、产业趋势、催化放在同一框架判断
- 判断周期行业是否进入反转区间

## 评分与概率的表达口径 (去伪精确, 2026-05-30)

> **动机**: 评分到 0.01 (7.40 vs 6.35)、概率到百分位 (25/50/25)、期望贡献到 0.1pp,
> 底层输入全是主观拍的。给主观判断套两位小数 = 制造"我很确定"的幻觉, 不增加准确性。

```
评分 (维度分 / 综合分):  只用 0.5 档 (…6.5 / 7.0 / 7.5 / 8.0…), 禁止两位小数
情景概率:               只用粗档 高/中/低 (≈60/30/10 一类), 禁止"25/50/25"伪精确
期望贡献 / 期望年化:     只用区间 + 方向 (如"较高 / 中等 / 偏负"), 禁止"+8.0%"单点
估值 / 价值锚:           用区间 (¥30-36), 不用单点 (¥33.6); 现价/锚用一位小数

唯一允许精确的: 真实成交价、真实持仓数量、真实账户金额、真实涨跌幅 (测量值, 不是估计值)
```

**判断口径**: 评分差 < 0.5 视为"无差异"。不允许因"7.40 > 7.0 门槛 0.40"而触发动作 —
0.4 分在主观打分里是噪声, 不是信号。

## Core Logic

### 1. 主线先行

判断公司是否处于市场愿意持续交易的方向：

- 一级主线：AI、算力、边缘 AI、机器人、自动驾驶、半导体、智能终端
- 二级主线：工业自动化、电网升级、高端制造、工业软件、新能源关键环节
- 周期反转：光伏、锂电、养殖等
- 事件型 / 非核心：游戏内容、高股息、防御、困境反转

输出必含：`mainline_strength` (strong/medium/weak) / `mainline_reason` (收入和利润如何映射) / `future_china_fit` (核心、配套还是边缘) / `cycle_position` (周期行业必标: 下行/磨底/反转初期/景气上行)。

### 2. 基本面底座

预期差成立前提：公司底子没坏。

最低检查：营收趋势、利润修复可能、经营现金流、ROE/毛利率、资产负债率/商誉/存货/应收。

回避：主业持续下行 + 利润靠非经常项 + 现金流弱于利润 + 估值贵但市场已一致看好。

**事实 / 解读 / 趋势 三层必须分清**：`confirmed_facts` (已发生硬数据) / `current_narrative` (市场当下解读, 可能与事实相反) / `trend_3_to_6_quarters` (3-6 季度趋势)。事实不变，解读会变，趋势最稳。**单季数据是噪声，趋势才是信号**。

### 3. 预期差判断

不要只写"跌多了"。必须拆四层：市场当前在担心什么 → 哪些是短期/情绪/一次性因素 → 最新资料里有哪些边际改善证据 → 改善何时可能被市场重新定价。

行业逻辑 vs 公司 α 必须区分：行业逻辑成立吗 → 这家公司是行业里最先受益方吗 → 公司基本面已在反映这个逻辑吗。三层都成立才能 buy。常见陷阱：把行业 β 当公司 α。

周期反转必须看到龙头先行修复（扣非环比收窄 / 现金流转正 / 毛利率转正 / 第二曲线 / 价格止跌）至少 2-3 个，才是真左侧。

### 4. 最新资料优先级

```
1. 已披露年报 / 季报
2. 业绩预告 / 业绩快报
3. 最新公告、业绩会、投资者关系纪要、产品发布、订单信息
4. 最新预约披露时间
5. 再回头看历史财报和走势
```

显式区分：`confirmed_facts` / `latest_materials` / `market_worries` / `expectation_gap` / `catalysts`。

### 5. 催化时间

没有催化的预期差，很容易长时间不兑现。必须列出：`catalyst_date` / `catalyst_type` / `what_market_needs_to_see`。

## Decision Rules

| 决策 | 条件 |
|-----|-----|
| **优先加仓** | 主线强 + 基本面硬 + 市场预期偏低 + 边际改善 + 催化临近 (或周期磨底→反转初期且龙头先行修复) |
| **继续持有** | 公司质量高，但预期差不明显或估值不便宜 |
| **逢高减仓** | 基本面一般 + 预期差弱 + 仓位过重 + 重估逻辑不牢 |
| **回避或调出** | 主线弱 + 基本面恶化 + 预期差是幻觉 + 催化不足 |

## Position Sizing Rules（集中投资分级仓位）

### 总体风格：5±1 只持仓集中投资 (硬上限 6)

Munger / 段永平式集中投资，不是公募双十式分散。

**集中度硬约束 (治"伪集中"病):**

```
诊断: 9 只持仓 + 单主题 65% + 现金 2.5% = "伪集中"
      —— 既担集中风险, 又因太散抓不住任何一只完整涨幅, 两头不讨好。

硬规则:
  1. 持仓数量硬上限 = 6 只 (理想 4-5 只)。超 6 → "只减不增", 4 周内退回 ≤ 6。
  2. 新进必先退: 已满 6 只时, 任何新标的入场 = 同时退出 1 只 (退最弱的)。
     且新标的期望贡献必须 > 被替换标的, 否则不换 (P4 期望)。
  3. 资金集中: 前 3 大持仓 ≥ 60% 总仓。低于此 = 摊太薄, 向高信念标的归并。
  4. 退出看"相对最弱", 不看"绝对好坏": 6 只都不错时, 也砍掉相对最弱的腾位。

判断"该退谁": 每周日按 (评分 × 期望贡献) 给全部持仓排序, 末位 = 退出候选。
```

### 仓位分级上限

| 类别 | 单只上限 | 数量上限 | 准入门槛 |
|------|---------|---------|---------|
| **核心持仓** | 25% | 2-3 只 | V14 6 选 5（见下） |
| **普通持仓** | 12% | 2-3 只 | 评分 6-8 + 主线辅助 |
| **试错档** | 4% | 1-2 只 | 反转赌博 / 主线探索 |

### 核心持仓准入门槛（V14 - 6 选 5）

> **2026/5/11 修正**: 旧 "5 选 4" 含"持有 ≥ 3 个月"是 V11 时代 proxy 约束 (作用 ≈ -1.2% 年化, Net Negative)。V14 已有 R1/R4/R5/R9/R12 等精确证据/概率工具替代时间约束。详见 cases/ 中对应演进案例。

```
1. 主线绝对硬: 新质生产力前 3 大主线 + 自身是主线核心     [P5 约束]
2. 评分 ≥ 8                                            [P1 证据]
3. FCEM 散度 ≤ 20% (高置信估值)                         [P1 证据]
4. 现价/价值锚 ≤ 1.5 (非超溢价区)                       [P4 期望]
5. 期望贡献最高档 ≥ +5% (R9 主力档级别)                 [P4 期望] (取代旧"持有 3 月")
6. 愿意按目标仓位加仓 ✴ + 心理压力测试 -25% 不操作       [心理压力]

满足 5/6 即可升核心档 25%
```

✴ 第 6 条"信念测试": 区分"真信念"（跌反而想加仓）和"死扛"（跌了不加也不卖）。

**为什么散度 + 估值 + 期望贡献三者并列**: 散度低 = 估值可靠; 估值不溢价 = 收益空间; 期望贡献 ≥ +5% = 强信念。三者同时满足 → 升核心是数学上最优解, 不是情绪决定。

### 不变量（仓位约束 - 完整版见 `references/decision-rules.md` R11）

- 同一主线 ≤ 30%（按主线分类计算，详见 `references/quantitative-triggers.md`）
- 持仓数量 ≤ 6 只 (理想 4-5); 满 6 只时新进必先退 (退最弱)
- 前 3 大持仓 ≥ 60% 总仓 (资金集中, 不摊薄)
- 单次买入 / 卖出 ≤ 持仓 1/3（强制分批）
- 现金 ≥ 5%; 试错档总和 ≤ 10%
- 评分 < 7 不能进主仓位档（仅试错 4%）
- 6 个月内不准修改仓位上限数字本身

## 止盈纪律 (Take-Profit Discipline)

> **动机**: 「涨了想跑」是处置效应, 「跌了死扛」是沉没成本。止盈体系的核心不是预测顶部, 而是**用规则锁住利润, 同时不设天花板**。

### 三种止盈模式

| 模式 | 触发条件 | 适用对象 |
|------|------|------|
| **移动止盈** | 从持仓期最高点回撤 18-20% → 减半 | **仅**动量/主题型赢家 (浮盈 > 10%) |
| **保本止盈** | 跌破成本线 → 全出 | 所有赢家, 浮盈 < 10% 时必设 |
| **趋势破位止盈** | 关键支撑破位 + 领先指标恶化 → 减仓 | 有外部镜像的标的 (如易点→APP) |

### 移动止盈规则

```
1. 记录持仓期最高收盘价 (不是日内高点)
2. 回撤 18-20% 从最高点 → 减半仓
3. 最高点上移 → 止盈位跟随上移 (只上不下)
4. 止盈位一旦 > 成本线, 该线 = 保利硬线, 永不撤销
```

### 趋势破位止盈

区别于移动止盈 (纯价格回撤), 趋势破位看**量价+领先指标**:

```
卖出条件 (满足 2/3 即执行):
  1. 放量跌破关键均线 (20日/60日, 量 > 1.5× 日均)
  2. 领先指标同步恶化 (易点→APP; 天赐→WTI; FII→NVDA)
  3. 连续 3 日低点下移 + 反弹无量
```

### 止盈 vs 止损 vs 减仓 区分

| 动作 | 触发 | 量 |
|------|------|------|
| **止损** | 跌破止损价 | 全出 |
| **止盈** | 移动止盈/保本止盈/趋势破位 | 先减半, 观察 |
| **腾资金减仓** | 需要现金, 从最弱票整只清 | 全出 (R6c 第1顺位) |

> 详细规则见 `references/decision-rules.md` R6/R6b/R6c/R6d。

## Decision Framework Pyramid（决策框架金字塔 V14）

> 旧 22+ 条铁律 (V11-V13 累积) 收敛为本结构。演进历史详见 cases/ 中铁律演进案例。
> **Layer 2 (R0-R23) 的完整执行细节、代码块、实证案例全部在 `references/decision-rules.md`。本文件只保留 Layer 0/1 + 一句话索引。**

### Layer 0: 公理 (Axioms)

```
A1 [目的] 炒股的目的是长期最大化期望收益, 不是"挂到股票"/"踏中风口"/"证明判断对"
A2 [决策] 决策依据 = 期望年化收益 × 触发概率 (期望贡献), 不是单一价格点位或单一估值
A3 [约束] 行动幅度有硬上限, 单次决策不致命, 累积纠错可持续
```

### Layer 1: 5 大原则 (Principles) — 新问题先演绎到这 5 条

| # | 原则 | 一句话 | 解决问题 |
|---|------|--------|---------|
| **P1** | 证据 Evidence | 估值/预测/操作必须有证据链 | 拍脑袋 / 盲信券商 |
| **P2** | 概率 Probability | 永远三档情景 + 显式概率 | 单点预测 / 过度自信 |
| **P3** | 对称 Symmetry | 买入门槛 ≤ 卖出门槛 | 买严卖松 / 现金堆积 |
| **P4** | 期望收益 Expected Value | 仓位按"期望贡献"排序 | 价格导向 / 挂寂寞 |
| **P5** | 约束 Constraints | 行动幅度有硬上限 | 重仓 / 情绪 / 失控 |

### Layer 2: 规则一句话索引 (细节按需查 `references/decision-rules.md`)

```
P0 前置
  R0   研究第一步必跑 data_fetcher.py / fetch_stock_data.py, 不允许凭印象 (a-stock-data 优先)

P1 证据
  R1   FCEM 四源交叉验证, 不盲信券商
  R1b  历史用一手数据, 未来自主推演, 一致预期仅 sanity check
  R2   yoy ≥ +30% 必做 7 步可持续性分析
  R3   最新季报必查 + 双校验 + 评分对冲
  R3.1 三层利润 → 评分定档 (维度 2 用最深层); 会计幻觉过滤器见 deep-research.md 维度 2.5
  R13  Forward PE 三年视角 (Y+1/Y+2/Y+3 fwd PE 表)
  R14  bottom-up 实证 (≥3 份研报 + 4 个核心问题), 行业 PE 中枢表见 forecasting-model.md

P2 概率
  R4   Bear/Base/Bull 三档 + 显式概率 (默认 25/50/25)
  R5   FCEM 散度 > 30% → 标记不确定
  R15  同行最新季 alpha 判定 → Bull/Bear 概率调整硬证据
  R16  账户级浮亏护栏 (浮亏越深, 单只上限自动收缩)

P3 对称
  R6   买卖对称 + 让赢家跑 (赢家默认持有, 移动止盈代替固定止盈)
  R6b  止损对称 (移动止盈位上移到成本之上即保利硬线)
  R6c  "让盈利奔跑" vs "提现金减仓" 是两个独立决策 (腾资金优先退弱票, 不削赢家)
  R6d  评分减仓联动 (单版本降≥0.5→降档; 连续2降→≤8%; 跌破7.0→退出候选; 跌破6.5→T+3强制减仓)
  R7   减仓量级 ≤ 加仓量级 × 1.5
  R8   减仓必同步配套加仓预案 (释放-部署)

P4 期望收益
  R9   仓位按"期望年化 × 触发概率"排序
  R10  双锚定: 战略档 = max(价值锚×0.85, 现价×0.80); 价值锚仅作认知工具

P5 约束
  R11  仓位上限 (主线≤30% / 核心25%普通12%试错4% / ≤6只 / 前3大≥60% / 现金≥5%)
  R12  执行约束 (100股整数 / A区禁买 / 双触发止损 / 到点必买 / 决策成熟即执行)
  R17  起仓档双过滤 (基本面 × 技术面), 禁止仅凭 PE 高就等大回调
  R18  挂单类型 (限价单 vs 条件单 vs 提醒), 按今日触发概率选择
  R19  决策频率 (盯盘不限, 决策必过 5 项过滤)
  R20  沉没成本绝缘 (错过/亏损/已投入时间不影响新决策)
  R21  基本面陈述零捏造 (AI 必须查证文档/财报, 答不出来源即承认错误)
  R22  价值锚更新触发 (基本面变化才更新, 不看时间)
  R23  自动记账 + 基准对比 (用户给出总资产时自动记一笔, 不记反事实)
```

### 演绎使用法（新问题如何套用本框架）

```
遇到新决策困境 → 走 5 原则 (而不是查 R 规则清单):

Step 1: 是否有证据? (P1) → 没证据 → 不动作 / 补研究
Step 2: 是否三档情景? (P2) → 单点预测 → 重做
Step 3: 买卖对称? (P3) → 不对称 → 调整门槛
Step 4: 期望贡献最大化? (P4) → 价格档位思维 → 改算期望贡献
Step 5: 行动幅度合规? (P5) → 超限 → 砍量级

新问题先演绎到 5 原则, 演绎不通才考虑加规则。
"加规则" 必须证明: 原则演绎无法覆盖此场景。
```

> **澄清 (反转股别用错尺子, 2026-06-03, 非新增规则, 是 P4 用法说明)**:
> 周期反转 / 困境反转股 (cycle_position = 磨底→反转初期, 且左侧信号 ≥ 3 个) 用 **P4 期望贡献 (赔率 × 概率)** 判断,
> **不用成长股的"当期基本面底座 / PE"打分**。当期亏损不是否决项, **改善斜率 + 左侧信号 (扣非环比收窄/毛利率转正/价格止跌/现金流转正/第二曲线) 才是**。
> 反转股早期天然: 当期亏损、PB 看似贵、只有 1 个转正季度 —— 拿成长股 8 维静态尺子去量, 必然误杀 (实证: 华灿 ¥7 时左侧 4/5 命中却被打低分观望)。
> 这是已有 P4 原则的用法澄清, **不新增 R 编号**。

## 规则冻结 + 减法治理 (2026-05-30, 最高优先级)

> **诊断**: 框架从 V11 的 12 条铁律, 一路加到 R0-R23 + R3.1 + R14.1 + 8 维评分 + FCEM 四源 + RSI 九宫格。
> 几乎每条规则都来自单一案例 (n=1)。这是典型的**对历史噪声过拟合** (curve-fitting): 参数越多越贴合过去, 对未来越脆。

### 冻结令 (硬约束)

```
1. 自 2026-05-30 起, 6 个月内 (至 2026-11-30) 禁止新增任何"R 编号"规则。
   例外: 仅当 (a) 5 原则演绎不能覆盖 + (b) 已造成真实金钱损失 (非反事实)
        + (c) track_performance 台账能看到对应的真实业绩拖累, 才允许加。
   口头补丁 ("下次注意…") 不算规则, 不写入。

2. 新教训默认只进 cases/, 不进 SKILL.md 主规则。
   case 是"备查的故事", 规则是"每次都要跑的检查"。绝大多数教训属于前者。

3. 任何想加规则的冲动, 先问: "这是新原则, 还是旧原则的又一个案例?"
   是案例 → 写进 cases/, 不动主文档。
```

### 减法清单 (主动收缩, 季度执行一次)

```
每季度末跑一次"规则体检", 对每条 R 规则问 3 个问题:
  Q1: 过去 3 个月, 这条规则实际触发过几次? (0 次 → 候选删除)
  Q2: 触发时, 它真的改变了决策、且事后被证明改对了吗? (说不清 → 候选降级)
  Q3: 它能从 5 原则直接演绎出来吗? (能 → 删规则, 留原则)

降级路径: 主规则 → 参考清单 → cases/ → 删除
目标: 规则总数只减不增, 直到"5 原则 + ≤10 条规则"可被一个人凭记忆执行。

已识别的过拟合候选 (下次体检优先审查):
  - R3.1 (三层利润定档表 + 会计幻觉过滤器): 来自易点 1 例, 疑似过拟合
  - R14.1 (行业 PE 中枢分级表 7 类): 表格化主观 PE
  - 8 维评分权重: 权重无实证来源, 未验证预测力
  - FCEM "四源交叉验证": 四方法共享同一批输入, 非独立信源
体检时不是"立刻删", 而是"举证保留": 谁主张保留, 谁拿出 track_performance 真实证据。
```

## Behavioral Rules（情绪与行动纪律）

### 核心认知：判断对错不是关键，控制行动幅度才是

无论判断多有信心，**单次行动幅度都必须有硬上限**。

### 强制冷静期

| 情境 | 行动 |
|-----|------|
| 看到坏消息想割肉 | 强制等 3 个交易日再决定 |
| 看到主线爆发想追高 | 强制等回调 5%+ 再介入 |
| 单日浮盈/亏 > 总资产 1.5% | 当天禁止任何交易决策 |
| 想"梭哈"或"全清" | 单次行动 ≤ 持仓 1/3 |
| 财报披露 T-3 至 T+0 | 完全禁止操作（详见 `references/earnings-discipline.md`）|
| 标的止损出局后 | T+0~T+14 完全冷静期 |

### 同一情绪的两面

- 恐慌（"再不卖会更惨"）= 底部割肉; FOMO（"再不买会错过"）= 山顶接盘
- 修复方法：不是判断更准，是单次行动 ≤ 1/3

### 盯盘节奏

- 一天最多 3 次（开盘 / 午后 / 盘后），每次 ≤ 10 分钟; 用价格预警替代主动盯盘; 绝不在 10:00-13:30 频繁刷盘

> 执行纪律的完整展开 ("慢决策, 快执行" / 期望值检验 / 三档建仓执行 / 实证执行错误) 见 `references/decision-rules.md` 文末"执行纪律细则 (R12 展开)"。

不重复整理已确认旧财报。只增量补：新披露财报 / 新业绩预告 / 新公告 / 新催化 / 旧担忧是否被证伪 / 仓位变化。

## Notes

- AI 不是唯一方向，是 `新质生产力` 里当前最容易形成催化和兑现的方向之一
- 宏观事件窗口期与财报窗口期叠加时，取**更严**的纪律
- 研究时优先问"它是不是中国未来的重要方向"，再问"AI 是否在其中起催化作用"
- 周期反转不是和成长主线对立，是另一套打法，但必须有供需和盈利拐点证据
- "好公司" ≠ "有预期差"; "便宜" ≠ "能涨"
- 所有判断都要尽量绑定最新资料和明确时间点
